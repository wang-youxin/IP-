# Phase -1: 语义切分与场景环境分析 v1.1

> **定位**：在所有蒸馏分析之前，将原始素材切分为语义完整的独立场景单元，对每个场景做六维环境分析，构建**环境→行为预测模型**。
> **核心理念**：一个人的表达不是孤立文本——它发生在具体的时间、空间、情绪、关系中。不理解场景环境，就无法理解"他为什么在这个时刻说了这些话"；不建立环境→行为映射，就无法预测"他在什么环境下会产出什么"。
> **下游依赖**：Phase 0~5 的所有分析都基于切分后的场景文档，而非原始素材。
> **v1.1 新增（2026-06-28）**：Step -1.4 环境→行为预测模型——环境聚类→行为映射→预测规则→最佳/最差环境配置→受众交互指南。

---

## 总览

```
原始素材（整段对话/整篇文章/整场直播）
       ↓
Step -1.0: 语义切分 ─── 按场景/话题/情绪/时间/对话回合切断
       每个切分点 = 一个语义完整的独立场景
       ↓
Step -1.1: 场景打标 ─── 每个场景分配：标题+场景ID+核心内容一句话
       ↓

       ↓
输出: N个场景文档（每个=一个独立.md）+ 场景地图
       ↓
进入 Phase 0: 素材诊断（基于场景文档，而非原始整段素材）
```

# Step -1.0: 语义切分

## 目的

将原始素材（一段45分钟直播、一篇5000字文章、一场2小时访谈）切分为语义完整的独立场景单元。

**切分粒度**：一个场景 = 一个完整的意思单位。太碎失去上下文，太大失去分析精度。

## 切分规则（5条触发条件）

按优先级排列。任一条件触发即切分：

### 规则1: 场景/环境变换（最优先）

| 变化信号 | 示例 | 切分动作 |
|---------|------|---------|
| 物理位置变化 | "好了我们接下来去..." / 画面切换 | ✅ 切分 |
| 人员变化 | 有人加入/离开/换人连麦 | ✅ 切分 |
| 媒介变化 | 从讲PPT切换到自由问答 / 从看屏幕切换到看镜头 | ✅ 切分 |
| 时间明显跳跃 | "三天后..." / "后来我们又..." / 剪辑痕迹 | ✅ 切分 |

### 规则2: 话题/主题切换

| 切换类型 | 判断标准 | 切分动作 |
|---------|---------|---------|
| **硬切换** | 明确话题转换词（"好，下一个问题""换个话题""说回刚才的"） | ✅ 切分 |
| **软切换** | 话题自然漂移，当前话题占比<20%已切换到新话题 | ✅ 在漂移完成处切分 |
| **回切** | 回到之前讨论过的话题（"刚才说到X，我再补充"） | ✅ 切分（标注"回切：话题X"） |
| **不切** | 同一话题下的子话题展开（举例/类比/深入） | ❌ 不切 |

### 规则3: 对话回合

| 信号 | 判断标准 | 切分动作 |
|------|---------|---------|
| 提问→回答链结束 | 一个完整QA从提问到回答完毕 | ✅ 在下一提问开始前切分 |
| 对方长段叙述 | 对方连续说>2分钟或>500字 | ✅ 切分为独立场景（标注"输入场景"） |
| 此人长段回应 | 此人连续说>3分钟或>800字 | ⚠️ 检查内部是否有话题切换（规则2），没有再保持完整 |
| 多人同时说话/打断 | 无序对话段 | ✅ 整体切分 + 标注"混乱场景" |

### 规则4: 情绪/能量突变

| 信号 | 判断标准 | 切分动作 |
|------|---------|---------|
| 情绪明显转变 | 平和→激动、认真→调侃、冷静→愤怒 | ✅ 在情绪转换处切分 |
| 能量水平突变 | 语速/音量突然变化 | ✅ 切分 + 标注能量变化 |
| 沉默/停顿 | 明显的长时间停顿（>5秒）且之后话题或情绪变化 | ✅ 在停顿后切分 |
| **不切** | 情绪在同类内波动（如从严肃→更严肃） | ❌ 不切 |

### 规则5: 叙述结构（针对文章/书籍）

| 信号 | 判断标准 | 切分动作 |
|------|---------|---------|
| 章/节/小节 | 明显的章节分隔 | ✅ 在章节边界切分 |
| "首先...其次...最后..." | 列表式结构 | ✅ 每一项切分 |
| "举个例子" | 案例展开 | ⚠️ 如果案例>200字，切分；否则不切 |
| "总结一下" | 总结段 | ✅ 切分 |
| 引言/结论 | 文章的开头和结尾 | ✅ 切分 |

## 切分操作规范

### 切分粒度标准

| 素材类型 | 目标场景数 | 单场景理想长度 | 过短/过长处理 |
|---------|----------|-------------|-------------|
| 直播/连麦（45min） | 8-15个场景 | 2-5分钟 | 过短(<1min)→与前合并；过长(>8min)→检查内部是否需要再切 |
| 深度访谈（1h） | 10-20个场景 | 2-5分钟 | 同上 |
| 文章/博客（5000字） | 5-10个场景 | 300-800字 | 过短(<100字)→与前合并；过长(>1500字)→再切 |
| 书籍章节 | 3-8个场景/章 | 500-1500字 | 按节切分 |
| 社交媒体（500条） | 按天/话题聚类 | 50-200字/条 | 单条不切，按天或话题聚类为场景 |

### 切分时的铁律

```
1. 不修改原文 — 切分只在原文边界处，不在句子中间切断
2. 保留上下文 — 每个场景文档内保留前后各1句话作为上下文铺垫
3. 重叠允许 — 关键过渡段落可同时属于前后两个场景（标注"过渡段"）
4. 完整性声明 — 记录切分决策：哪些地方切了/为什么/哪些地方没切但可能有争议
5. 可还原 — 按场景编号顺序拼接 = 原始素材（无丢失）
```

## 输出格式

每个切分后的场景生成一个独立 `.md` 文件：

```
02-场景文档库/
├── _场景地图.md                    ← 所有场景的索引地图
├── _切分决策日志.md                 ← 记录每次切分的理由和争议
├── SRC-XX/                        ← 按原始素材分组
│   ├── SRC-XX-001/
│   │   ├── SCN-XX-001-01.md       ← 该素材的第1个场景
│   │   ├── SCN-XX-001-02.md       ← 该素材的第2个场景
│   │   └── ...
│   ├── SRC-XX-002/
│   └── ...
```

# Step -1.1: 场景打标

## 目的

为每个切分好的场景分配元数据：唯一ID、标题、核心内容一句话、场景类型标签。

## 场景ID规则

```
SCN-[人物缩写]-[素材序号]-[场景序号]

示例: SCN-LDM-001-03 = 鲁大魔的第1段素材中的第3个场景
```

## 打标模板

每个场景文档的 frontmatter：

```yaml
---
id: SCN-XX-XXX-XX
source: SRC-XX-XXX                    # 来源素材ID
source_position: "12:34-18:20"        # 在原始素材中的时间/位置
title: "[此场景的一句话标题]"
scene_type: [对话问答|单人讲述|实战诊断|争论冲突|闲谈互动|演讲展示|写作输出|复盘反思|其他]
core_content: "[此场景的核心内容——一句话说清楚这个场景在讲什么]"
keywords: [关键词1, 关键词2, 关键词3]
participants: [此人角色, 对方角色/身份]
word_count: N
created_at: YYYY-MM-DD
---
```

### title 命名规则

```
优先级:
1. 此人自己总结的 → 用他的话（"'做IP不是做内容，是做资产'—鲁大魔2024-03-15直播"）
2. 场景的核心冲突/问题 → 问句式（"为什么努力的人反而赚不到钱？"）
3. 场景的核心动作+对象 → 动宾式（"诊断一个想做个人IP的餐饮老板"）
4. 都不适用 → 描述式（"关于流量衰减的即兴分析"）

禁止: 空洞标题（"对话片段3"）、术语堆砌（"归因翻转+变量压缩+不对称回应"）
```

### scene_type 分类

| 类型 | 定义 | 典型场景 | 蒸馏价值 |
|------|------|---------|---------|
| **对话问答** | 对方提问→此人回答 | 连麦诊断、Q&A环节、评论回复 | ⭐⭐⭐ 暴露即兴思维+诊断方法 |
| **单人讲述** | 此人独立输出观点 | 直播单口、课程讲解、写文章 | ⭐⭐⭐⭐ 系统化思考+核心主张 |
| **实战诊断** | 此人正在解决一个具体问题 | 现场分析案例、帮人看项目 | ⭐⭐⭐⭐⭐ 最真实的方法论展示 |
| **争论冲突** | 有不同意见的交流 | 与人对谈有分歧、回应批评 | ⭐⭐⭐⭐⭐ 揭示核心信念+底线 |
| **闲谈互动** | 非正式的随意交流 | 直播闲聊、评论区互动、私下聊天 | ⭐⭐ 揭示性格侧面+口头禅 |
| **演讲展示** | 精心准备的正式输出 | Keynote、发布会、课程 | ⭐⭐⭐ 精心设计的信息+表达DNA |
| **写作输出** | 系统性书面表达 | 文章、书稿、newsletter | ⭐⭐⭐⭐ 最系统的思维展示 |
| **复盘反思** | 回顾过去的决策/行为 | "后来我想了想..."、"我犯过一个错..." | ⭐⭐⭐⭐⭐ 自我认知+思想演进 |

# Step -1.2: 场景环境分析（六维分析）

> **核心问题**：这个场景是在什么环境下发生的？这个环境如何影响了此人的表达和行为？

## 六维分析框架

### 维度1: 🏠 物理环境

```markdown
### 物理环境
- **空间**: [直播间/办公室/家里/户外/演播室/咖啡厅/车里/未知]
- **时间**: [具体时间/时段（凌晨/上午/下午/深夜）]
- **时长**: [此场景持续多久]
- **媒介**: [面对面/视频连麦/语音/文字/录播]
- **设备**: [手机/电脑/专业设备/未知]
- **环境特征**: [嘈杂/安静/有背景音/有BGM/有他人在场]
- **此人舒适度**: [熟悉环境/陌生环境/不确定] — 判断依据: [如"在自己直播间，言行放松"]
```

### 维度2: 👥 社交环境

```markdown
### 社交环境
- **对方身份**: [粉丝/客户/同行/朋友/记者/学生/公众/无人(独白)]
- **对方与此人关系**: [陌生人/熟人/老朋友/对手/学生/导师/平等]
- **权力关系**: [此人高于对方/对等/此人低于对方] — 判断依据: [如"对方是付费咨询客户，此人处于服务方"]
- **对方状态**: [困惑/质疑/求助/挑衅/学习/闲聊/情绪化]
- **有无观众**: [无/有(N人)] — 观众身份: [粉丝/同行/路人]
- **一对一/一对多**: [一对一对话/一对多输出/多对多]
- **社交压力**: [低/中/高] — 判断依据: [如"直播中，有观众，此人知道被公开记录"]
```

### 维度3: 💭 情绪场

```markdown
### 情绪场
- **此人开场情绪**: [平静/兴奋/疲惫/愤怒/幽默/严肃/焦虑/不耐烦/热情/冷淡]
- **此人情绪变化**: [稳定/先X后Y/波动] — 变化触发点: [什么时候情绪变了？]
- **对方情绪**: [与上同格式]
- **情绪互动模式**: [共鸣型(情绪一致)/对比型(此人保持冷静对方激动)/引导型(此人主动调节对方情绪)/对抗型(情绪对立)]
- **情绪强度**: [1-5] — 1=完全平静, 5=极度激动
- **情绪真实性**: [自然流露/刻意表现/不确定] — 判断依据: [如"被质疑后语速加快+音量提高，不像表演"]
```

### 维度4: 🔫 触发条件

```markdown
### 触发条件
- **场景触发方式**: [对方提问/此人主动发起/事件驱动/承接上一场景/随机展开]
- **触发问题/事件**: [具体是什么触发了这个场景？一句话描述]
- **对方原话**（如果是提问触发）: "[对方原话]" — SRC追踪
- **此人是被动回应还是主动输出**: [被动(回应对方)/主动(想表达)/混合(借对方的提问展开自己的观点)]
- **此人的回应策略**: [直接回答/反问/绕开/借题发挥/部分回答/拒绝回答]
```

### 维度5: ⚡ 压力等级

```markdown
### 压力等级
- **压力来源**: [时间压力/观众压力/对方质疑/情绪对抗/知识盲区/道德困境/多任务/无压力]
- **压力等级**: [1-5] — 1=完全放松, 5=极限压力
- **压力信号**: [语速变化/音量变化/回避/反问增多/沉默/用通用词代替具体词/转移话题/其他]
- **此人面对压力的模式**: [迎上/回避/转化/崩溃/不变] — 判断依据: [如"对方质疑数据，此人直接反问'你说的数据来源是什么'——迎上型+不进入对方框架"]
- **压力是否影响表达质量**: [是/否/反而更好] — 判断依据: [如"在有压力的追问下，逻辑反而更清晰"]
```

### 维度6: 🎭 角色状态

```markdown
### 角色状态
- **此人扮演的角色**: [诊断者/老师/同行交流者/内容创作者/朋友/销售/思考者/防御者/表演者]
- **角色一致性**: [与此人主要类型一致/不一致(扮演了不常见的角色)]
- **角色切换**: [此场景中角色是否切换？] — 从[X]→[Y]，触发点: [什么导致切换]
- **角色舒适度**: [舒适/勉强/不舒适] — 判断依据: [如"被要求给出具体承诺时从'老师'切换到'防御者'"]
```

## 六维综合分析

```markdown
### 六维综合分析

**场景一句话画像**:

- 影响方向: [促进型(环境让他更好)/抑制型(环境让他更差)/无明显影响]

- 对比: 如果知道此人在不同环境下的同类表达，标注差异: [如"此人在私下对话中对同一话题的表达更谨慎——对比SCN-XX-XXX-XX"]

**此场景的独特价值**:
```

# Step -1.3: 场景关系初步建立

## 目的

场景不是孤立的——它们之间存在时间顺序、因果关系、情绪演变。在逐场景分析之前，先建立场景之间的宏观关系。

## 场景关系类型

| 关系类型 | 定义 | 示例 |
|---------|------|------|
| **时间顺序** | A发生在B之前 | SCN-001-01 → SCN-001-02 |
| **因果链** | A直接导致了B | 对方质疑(SCN-001-03) → 此人反击(SCN-001-04) |
| **情绪演变** | A的情绪状态发展到B | 平静讨论→逐渐激动→爆发 |
| **话题延续** | B是A的同一话题的深入/展开 | "刚才说的第一点我展开..." |
| **结构对应** | A和B是同一结构的正反/并列 | 正面案例→反面案例 |
| **独立场景** | 与前后无直接关系 | 中场休息闲聊 |

## 场景地图格式

```markdown
## 场景地图 — [人物名] — [素材名: SRC-XX-001]

### 场景时间线
```
SCN-001-01 [对话问答] "为什么我的账号做不起来？" (12:34-15:20)
  ↓ 情绪: 对方焦虑→此人安抚
SCN-001-02 [单人讲述] "做内容不是做数据，是做信任" (15:20-18:45)
  ↓ 此人从回应→升华为方法论。场景01触发，情绪从安抚→激昂
SCN-001-03 [对话问答] "那我具体怎么开始？" (18:45-22:10)
  ↓ 回到具体操作。情绪: 平静+具体
SCN-001-04 [闲谈互动] 聊到最近的热点事件 (22:10-25:00)
  ↓ 话题跳转。情绪: 放松吐槽
SCN-001-05 [实战诊断] 看对方的账号首页分析 (25:00-32:00)
  ↓ 进入诊断模式。情绪: 专注+犀利
...
```

### 场景关系矩阵
|  | SCN-01 | SCN-02 | SCN-03 | SCN-04 | SCN-05 |
|--|--------|--------|--------|--------|--------|
| SCN-01 | — | 因果 | 因果 | 无关 | 无关 |
| SCN-02 | 被引发 | — | 话题延续 | 无关 | 铺垫 |
| SCN-03 | 被引发 | 深入 | — | 无关 | 无关 |
| SCN-04 | 无关 | 无关 | 无关 | — | 无关 |
| SCN-05 | 无关 | 被铺垫 | 无关 | 无关 | — |
```
