# Part A: 三条标准逐条对标

## 标准1: 原文一定要保存好，一定要可溯源

### v2.0 已有的 ✅

| 机制 | 位置 | 状态 |
|------|------|------|
| 素材注册表 (SRC-ID) | Step 0.0 | ✅ 定义了 |
| source_documents 字段 | 附录C YAML标准 | ✅ 定义了 |
| source_quotes 字段 | 附录C YAML标准 | ✅ 定义了 |
| 溯源铁律（4条） | Step 0.0 + Phase 1开头 | ✅ 声明了 |

### v2.0 缺失的 ❌ — 5个关键缺口

#### 缺口1: 溯源铁律只声明了，没有在执行层嵌入

**问题**：Step 0.0声明了4条溯源铁律，Phase 1开头重复了一次。但Phase 1.1-1.11和Phase 2.1-2.7的**每个提示词内部**没有溯源检查步骤。

**后果**：AI执行提示词时，如果提示词本身不要求引用SRC-ID，就不会引用。铁律变成"贴在墙上的口号"。

**修复**：每个Phase维度提示词的**输出格式中**，必须内嵌溯源检查项。不是放在"质量检查"里——是放在输出模板中。例如：

```
❌ v2.0 (溯源在外部):
  Step 1.1的提示词要求输出"高频词Top 20"
  溯源要求只在Phase 1开头的"铁律"段落

✅ v3.0 (溯源内嵌):
  Step 1.1的提示词要求:
  - 高频词Top 20（必须附原始计数 + SRC-ID: "[原文例句]" — SRC-XX-XXX）
  - 每个自创术语标注首次出现SRC-ID
  - 每个口头禅标注频率统计（N次/万字，基于SRC-XX-001至SRC-XX-050的全文统计）
```

#### 缺口2: 原文保存格式标准缺失

**问题**：Step 0.0说了"保存素材副本"，但没有定义格式标准。视频转文字的格式是什么？时间戳精度是多少？社交媒体导出用什么格式？

**后果**：不同人执行时，原文保存格式不一致，后续溯源无法精确定位。

**修复**：新增 `原文保存格式标准`：

| 素材类型 | 保存格式 | 时间戳精度 | 必需元数据 |
|---------|---------|----------|----------|
| 视频→文字 | SRT + 全文transcript (.md) | 精确到秒 (MM:SS) | 日期、时长、标题、来源URL |
| 音频→文字 | transcript (.md) | 精确到秒 | 日期、时长、标题 |
| 书籍/PDF | 章节级引用 (章.节.段) | 不适用 | 书名、版本、ISBN、页码 |
| 社交媒体 | JSON导出 + 时间戳 | 精确到分钟 | 平台、账号、导出日期 |
| 网页/博客 | Markdown + URL + 抓取日期 | 不适用 | URL、抓取日期、作者 |
| 直播录屏 | SRT + transcript | 精确到秒 | 日期、时长、平台、参与者 |

#### 缺口3: 原文引用的颗粒度没有分级

**问题**：附录C定义了 `"[原文]" — SRC-XX-003` 格式，但没有区分引用颗粒度。一段45分钟的对话，只引用"SRC-XX-003"不够——读的人找不到原文位置。

**修复**：新增三级引用颗粒度：

| 级别 | 格式 | 何时用 | 示例 |
|------|------|--------|------|
| **L1: 素材级** | SRC-XX-003 | 引用整段素材的背景/语境 | 用于宏观判断 |
| **L2: 位置级** | SRC-XX-003, 12:34-12:58 | 引用对话/视频中的具体段落 | 用于观点提取 |
| **L3: 精确级** | SRC-XX-003, 12:34, "[原文字句]" | 引用关键判断/定义/转折 | 用于CON/OPI定义 |

**规则**：CON的定义必须≥L2。OPI的提取≥L2。CAS的提取≥L2。任何声称"此人的核心信念"的结论必须≥L3。

#### 缺口4: 原文转述的"退化链"没有追踪

**问题**：素材→CON→OPI→Skill 的过程中，原文可能经过多次转述。目前没有机制追踪"这个结论离原文有几步"。

**后果**：最终的Skill中的一句话可能是从原文经过了3层转述，每一层都可能引入偏差。

**修复**：新增 `转述距离标记`：

```
L0: 原文直接引用（"[此人原话]" — SRC-XX-003）
L1: 一次转述（分析者用自己的话概括此人原话，附原文对照）
L2: 二次抽象（从L1概括中进一步抽象，标注"基于L1概括"）
L3: 综合推断（从多个L1/L2中综合判断，标注"综合推断"）
```

**规则**：Skill的"心智模型"部分最多接受L2。L3必须进入诚实边界。

#### 缺口5: 没有原文保存的"完整性声明"

**问题**：Step 0.1素材清点列出了有什么素材，但没有声明"我们缺什么"。用户可能以为蒸馏结果是基于完整素材的，实际上关键素材缺失。

**修复**：Step 0.1输出中新增 `素材完整性声明`：

```markdown
### 素材完整性声明
- 已覆盖的素材类型: [N]/[11]类
- 缺失的素材类型: [列表]
- 此人公开发言总量估计: [X小时]
- 已获取: [Y小时] = [Z%]
- 关键缺失: [如果他有过X但未获取，列出]
```

## 标准2: 底层方法论一定要够底层且有大量案例可交叉验证

### v2.0 已有的 ✅

| 机制 | 位置 | 状态 |
|------|------|------|
| Phase 2 统计验证要求 | Step 2.1/2.2 | ✅ 频率统计+验证案例+反例 |
| 三级质量分层 (OPI) | Step 1.7 | ✅ ★★★/★★☆/★☆☆ |
| 推导链要求 | Step 2.2 | ✅ 如果X→因为Y→所以Z |

### v2.0 缺失的 ❌ — 6个关键缺口

#### 缺口6: Phase 1 没有统计验证要求

**问题**：Phase 2有严格的统计验证（频率≥60%、≥3验证案例、≥2反例），但Phase 1的11个通用维度没有同等级别的要求。表达DNA的高频词统计要求了"原始计数"，但没有要求验证案例。智识谱系没有要求"每个影响源标注此人引用的具体次数"。内在张力的每对矛盾没有要求统计双方各出现多少次。

**后果**：底层方法论（Phase 2）有根基，但中层分析（Phase 1）可能建立在不可靠的数据上。

**修复**：Phase 1每个维度新增统计验证要求：

| Phase 1 维度 | 新增统计要求 |
|-------------|------------|
| 1.1 表达DNA | 每个句式偏好配≥10条例句（而非3-5条），句式分布必须有百分比（而非"为主"），高频词统计覆盖全部素材 |
| 1.2 智识谱系 | 每个"明确提及"来源标注引用次数（而非只列名字），推测影响标注与其他独立来源的交叉验证 |
| 1.3 时间线 | 每个转折点标注"前后立场的证据数"，核心信念标注"跨年度的重申次数" |
| 1.4 他者视角 | 每条批评标注"来自几个独立来源"，合理性评估标注评判依据 |
| 1.5 内在张力 | 每对矛盾标注双方在各出现多少次（A出现N次，B出现M次），标注矛盾的稳定性（加深中/减弱中/稳定） |
| 1.6 概念体系 | 每个CON标注"在素材中出现的总次数"，典型应用≥5个（而非3个） |
| 1.7 观点库 | 保持三级分层，每个观点标注"在不同素材中被重复的次数" |
| 1.8 案例库 | 每个案例标注"此人重复引用此案例的次数"（重复引用=更重要的案例） |

#### 缺口7: "底层方法论"缺乏跨人物验证

**问题**：Phase 2从一个人身上提取的方法论（如"不对称回应律"），没有机制验证它在其他人身上是否也成立。如果它只在一个人身上验证过，不能叫"底层方法论"——只能叫"此人的行为模式"。

**后果**：框架声称产出了"底层方法论"，但实际上只是"个人习惯的统计描述"。

**修复**：新增 `方法论深度评级`：

| 深度级别 | 验证要求 | 可以声称 |
|---------|---------|---------|
| **L0: 个人习惯** | 仅在1人身上出现≥3次 | "此人的行为模式" |
| **L1: 个人规则** | 1人+≥60%频率+≥3案例+推导链 | "此人的底层运算规则" |
| **L2: 类型模式** | 同类型≥3人身上都出现 | "诊断型人物的通用模式" |
| **L3: 跨类型模式** | ≥2种类型≥5人身上都出现 | "人类通用底层思维规则" |

**规则**：蒸馏框架默认产出L1。不能声称L2/L3除非有跨人物数据。诚实地标注每个"规则"的深度级别。

#### 缺口8: 缺乏反例的系统化收集机制

**问题**：Phase 2要求"≥2个反例分析"，但这是被动的——假设反例已经被素材覆盖。实际上，很多人物的素材中只会展示成功的应用，不会展示失败的案例。

**后果**：提取的方法论可能是"幸存者偏差"——只看到此人用这个方法成功的场景，看不到失败或不适用的场景。

**修复**：新增 `反例挖掘策略`：

1. **主动搜索**：在此人的素材中搜索"我错了一次""当时不该用""这不适用于"等自述失败
2. **逻辑反推**：如果方法A适用于条件X，那么条件非X时，方法A应该不适用——检查有无反例
3. **外部验证**：搜索他人对此人失败决策的分析/批评（如果他者视角中有"那次他用错了方法"）
4. **标注反例质量**：
   - 🟢 直接反例：此人明确承认或可观察到的方法失效
   - 🟡 推理反例：逻辑上应该存在但未观察到的反例
   - 🟠 缺失标注：无法确认是否存在反例（素材不足）

#### 缺口9: 推导链的质量标准缺失

**问题**：Step 2.2要求每条规则有推导链（如果X→因为Y→所以Z），但没有定义什么算合格的推导链。一个不成立的推导链比没有推导链更危险。

**后果**：AI可能生成看起来合理但逻辑不通的推导链。

**修复**：新增 `推导链质量检查`：

```
合格的推导链 = 三个环节全部可验证
1. 如果[X] → X必须是从素材中可直接观察到的触发条件（有原文证据）
2. 因为[Y] → Y必须是此人的核心信念/CON，且有独立证据（不能只用推导链本身来证明Y）
3. 所以[Z] → Z必须是从素材中可直接观察到的行为结果（有原文证据）
4. 替代解释 → 有没有其他解释也能解释"为什么X→Z"？如果有，标注

不合格的推导链示例（会被打回）:
- "如果他觉得对方不自知 → 因为他重视效率 → 所以他会直接打断"
  问题: "重视效率"没有独立证据，只是从"直接打断"反向推测的（循环论证）
```

#### 缺口10: 没有"方法论的可证伪性"检查

**问题**：框架提取的方法论都是描述性的（"此人在X情况下做Y"），没有定义"什么观察能证明这个方法论是错的"。

**后果**：方法论变成不可证伪的——无论看到什么行为，都能"解释"为符合方法论。

**修复**：每个方法论必须附加 `可证伪条件`：

```markdown
### 方法1: [名称]
- 规则: 如果[X]，则此人会用[Y]
- **可证伪条件**: 如果在≥3次观察到[X]的情况下此人没有用[Y]，且无法用失效条件解释 → 此规则被证伪
- **已证伪**: 是/否（如果"是"→此规则进入纠错日志）
```

## 标准3: 100% 双链连接

### v2.0 已有的 ✅

| 机制 | 位置 | 状态 |
|------|------|------|
| 双链验证门禁 (死链检查) | Step 2.9b | ✅ |
| 双链完整性终检 | Step 5.3 | ✅ |
| 5种关系类型 | Step 1.9a | ✅ |
| Obsidian双链格式 | 附录C | ✅ |

### v2.0 缺失的 ❌ — 4个关键缺口

#### 缺口11: 双链只有单元级，没有段落级和证据级

**问题**：目前的双链是CON↔OPI↔CAS之间的链接。但一个CON内部的不同段落，可能引用不同的素材、关联不同的OPI。一个OPI的"原文引用"和"分析者概括"应该各自有链接。

**后果**：双链是粗粒度的。一个500字的CON文档中，只有文档级的frontmatter关系，但文章中具体哪句话来自哪个SRC，无法直接追踪。

**修复**：新增 `段落级双链标准`：

```
文档内引用格式（支持段落级定位）:
- 句级: 这句话引用了[原文] — SRC-XX-003, L3 (精确级)
- 段级: 本段基于 SRC-XX-003, 12:34-12:58 (位置级)
- 文档级: 本文的 source_documents: [SRC-XX-003, SRC-XX-007]

知识库产物要求:
- 每个.md文件中，至少70%的段落有段级或句级溯源标记
- Skill产物中，每个核心声明有至少L2级溯源
```

#### 缺口12: 溯源链不是100%覆盖，只是随机抽查

**问题**：Step 5.3终检说"随机抽取5条素材ID"和"随机抽取5个关键结论"进行反向追踪。这是抽查，不是全量验证。

**后果**：抽查通过的产物，可能有大量未被抽查到的死链或不可溯源结论。

**修复**：Step 5.3改为 `全量自动化验证 + 抽查人工验证`：

```
全量自动化检查（100%覆盖）:
1. 扫描所有文件的所有 [[链接]]，全部可解析 → 死链=0
2. 扫描所有 source_documents 字段，全部SRC-ID在注册表中存在
3. 扫描所有单元，计算素材覆盖率 = 被引用的SRC-ID数 / 总SRC-ID数
4. 扫描所有孤儿单元

随机人工深度验证（抽查5条）:
1. 随机抽5条SRC-ID，人工验证其原文确实支持引用的结论
2. 随机抽5个关键结论，人工追溯其完整溯源链（结论→OPI→SRC→原文）
```

#### 缺口13: 没有"动态双链"机制

**问题**：框架假设蒸馏是一次性的——所有素材在手，一次性蒸馏完。但实际上，活人还在不断产出新内容。当新素材加入时，已有的CON/OPI/CAS如何更新？如何确保新增内容的双链与旧内容一致？

**后果**：蒸馏产物是静态快照，随时间推移越来越不准。新素材加进来时，如果没有更新机制，双链会逐渐断裂。

**修复**：新增 `动态更新协议`：

```
当新素材加入时:
Step U1: 新素材注册（分配SRC-ID，追加到注册表）
Step U2: 新素材扫描——是否需要创建新的CON/OPI/CAS？
Step U3: 已有单元更新——新素材是否支撑/冲突/补充了已有的单元？→追加source_documents
Step U4: 关系更新——新素材是否建立了新关系？是否暴露了之前未发现的冲突？
Step U5: 双链重检——更新后重新运行Step 2.9b（死链检查）和Step 5.3（终检）
Step U6: 版本标记——被更新的单元增加 version 号，旧版本不删除（保留历史）
```

#### 缺口14: 没有"链接强度"分级

**问题**：目前的关系只有类型（回应/解释/证明/冲突/共享主题），没有强度。CON-001"证明"OPI-003，是强证明还是弱证明？这影响交叉分析中的权重。

**后果**：关系网络中，一条弱关系和一个强关系被同等对待，网络中心分析可能失真。

**修复**：新增 `关系强度标注`：

| 强度 | 含义 | 何时用 |
|------|------|--------|
| ★★★ 强 | 直接的、此人有明确表述的连接 | CON的直接依赖，OPI的明确回应 |
| ★★☆ 中 | 间接的、分析者推断的连接 | 推测影响、间接解释 |
| ★☆☆ 弱 | 松散的、仅共享主题的连接 | 共享主题、可能相关 |

# Part B: 6个结构性缺口

## 缺口15: Phase 0缺少"内容方向-覆盖面积-实际素材"三角判断

**问题**：你说的"内容方向是什么？覆盖的面积是什么？实际素材有什么？"——这是三个不同的问题，但现有Phase 0只回答了第三个（素材清点）。

**修复**：在Step 0.1和0.2之间，新增 **Step 0.15: 内容方向与覆盖面积判定**。

详见 [v3.0-Step0.15-内容方向与覆盖面积判定.md](./v3.0-Step0.15-内容方向与覆盖面积判定.md)

## 缺口16: 缺少参考人物库

**问题**：女娲skill有14个人物案例（Jobs, Musk, 费曼, 芒格, Naval, Taleb, Karpathy, Paul Graham, Ilya, MrBeast, Trump, 张一鸣, 孙宇晨, 还有一个X大师）。这些案例没有作为框架的"验证集"使用。

**后果**：每次蒸馏新人时，无法对标已有案例，无法说"这个人跟Jobs在X维度上的相似度是Y%"。

**修复**：新增 `参考人物库`：

详见 [v3.0-参考人物库.md](./v3.0-参考人物库.md)

## 缺口17: 缺少执行中的纠错和回退机制

**问题**：框架是线性的——Phase 0→1→2→3→4→5。但没有定义"如果Phase 3发现Phase 1的某个结论是错的，怎么办"。

**修复**：新增 `纠错回退协议`：

```
纠错触发条件:
- Phase N的发现与Phase N-1的结论冲突（例如：Phase 3发现的新案例与Phase 1的概念定义矛盾）
- 新素材暴露了之前未发现的错误
- 交叉分析发现了之前忽略的矛盾

回退流程:
1. 标注冲突点（哪个Phase的哪个结论与什么新发现冲突）
2. 评估影响范围（哪些下游单元依赖此结论？）
3. 回退到最早受影响的Phase，重新执行
4. 版本标记（被修正的单元增加version号）
5. 写入纠错日志（继承dbs-correction的格式）
```

## 缺口18: Phase 4交叉分析太弱

**问题**：目前只有6个交叉维度，全部是两两交叉。缺少三重交叉和全维度综合评分。而且没有"交叉分析发现了新问题后怎么办"的后续机制。

**修复**：新增三重交叉 + 综合评分：

```
Phase 4 升级:
4.1 两两交叉（保持现有的6个）
4.2 三重交叉（新增）:
  - 概念×时间线×智识谱系: 每个概念的思想来源+出现时间 → 思想演进的三维地图
  - 表达DNA×内在张力×他者视角: 语言特征在矛盾时的变化+外部是否注意到了
  - 操作方法×底层规则×价值观: 操作→规则→价值的层级推导是否自洽
4.3 全维度综合评分（新增）:
  - 思想体系自洽度: [%] （内在矛盾数/总观点数）
  - 言行一致度: [%] （有决策记录时）
  - 蒸馏置信度: [%] （基于一手占比+时间跨度+交叉验证强度）
  - 方法论通用度: [L0/L1/L2/L3]
4.4 交叉分析发现的新问题 → 回退协议（缺口17）
```

## 缺口19: 诚实边界不是"活的"

**问题**：Step 1.11在Phase 1结束时做诚实边界，Step 4.2做汇总。但Phase 2和Phase 3的新发现不会主动回补到诚实边界中。

**修复**：诚实边界改为 `持续更新的活文档`：

```
诚实边界更新节点:
- 每个Phase结束时，检查是否有新增局限 → 追加到诚实边界
- Phase 4交叉分析发现的矛盾/不确定 → 追加到诚实边界
- Phase 5产物生成时，诚实边界随产物一同输出

诚实边界的结构升级:
1. 素材局限（来自Phase 0）
2. 方法论局限（来自Phase 1-3各维度的局限汇总）
3. 交叉分析局限（来自Phase 4发现的不确定）
4. 动态局限（来自新素材加入后的更新）
5. 使用局限（来自产物验证）
6. 提升路径（来自所有以上）
```

## 缺口20: 产物质量验证标准不够

**问题**：Phase 5生成了Skill或知识库，但没有定义"好的产物"和"坏的产物"的判断标准。Step 5.3的双链终检是必要的但不充分的——双链完整≠内容准确。

**修复**：新增 `产物质量验证四维度`：

```
产物质量 = 双链完整度 × 溯源覆盖率 × 方法论验证度 × 使用有效性

维度1: 双链完整度 (Step 5.3终检)
  - 死链数=0 ✅
  - 素材覆盖率≥70% ✅

维度2: 溯源覆盖率 (新增)
  - 所有核心声明可追溯到原文: N/M = X% → 目标≥90%
  - 转述距离≤L2的占比: X%

维度3: 方法论验证度 (新增)
  - 每个方法有≥3验证案例的占比
  - 每个规则有≥2反例分析的占比
  - 推导链合格率

维度4: 使用有效性 (新增)
  - 盲测: 让蒸馏产物回答N个此人的真实被问过的问题
  - 对比: 蒸馏产物的回答 vs 此人的真实回答（相似度评分）
  - 失败模式记录: 在什么问题类型上蒸馏产物给出了不符合此人的回答
```

## 一句话总结

**v2.0框架的方向是对的，三条标准的意识是有的。但执行层有三个根本问题：①溯源是"贴在墙上的口号"而非"嵌入每个提示词的引擎"，②底层方法论没有跨人物验证机制，③双链只有粗粒度单元级。**

**v3.0的目标：把口号变成引擎，把单人验证变成跨人验证，把单元级双链变成段落级双链。**
