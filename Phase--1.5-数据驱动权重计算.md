# Part 1: 七Head权重计算

## Head 1: 词频权重 (TF-W, Term Frequency Weight)

### 目的
统计每个关键词在所有场景中的出现频率——最基本的权重信号。

### 计算方式

```
TF-W(k) = log(1 + tf(k)) / log(1 + max_tf)

其中:
  tf(k) = 关键词k在所有场景中出现的总次数
  max_tf = 所有关键词中最高出现次数
  log压缩防止少数极高频词主导
```

### 特殊处理

```markdown
| 情况 | 处理 |
|------|------|
| 功能性高频词（"这个""然后""就是"） | 停用词过滤，不参与权重计算 |
| 此人的口头禅/高频词（"破锁""归因翻转"） | 加权——口头禅=此人自己定义的概念 |
| 通用术语但在此人语境下特殊（"流量""资产"） | 标注"语境依赖"，需配合共现权重确认 |
```

### 输出格式

```markdown
### TF-W Top 20
| 排名 | 关键词 | 出现次数 | 出现场景数 | TF-W |
|------|--------|---------|----------|------|
| 1 | "IP资产" | 47 | 18/25 | 0.92 |
| 2 | "复利思维" | 38 | 15/25 | 0.87 |
| 3 | "信任前置" | 35 | 14/25 | 0.85 |
| ... | ... | ... | ... | ... |
```

## Head 2: 共现权重 (CO-W, Co-occurrence Weight) — 最核心

> **类比 Transformer**: 这相当于 Self-Attention 的核心部分——一个词的重要性不在于它自己出现了多少次，而在于它和哪些词一起出现。

### 目的
通过关键词共现关系图，计算每个关键词的网络中心性。

**直觉**: 一个词如果和很多其他重要词一起出现，它可能是"枢纽概念"——即使它自己出现频率不高。

### 计算步骤

#### Step 1: 构建共现矩阵

```
对每个场景:
  提取该场景的关键词列表 [k1, k2, k3, ...]
  对列表中任意一对关键词 (ki, kj):
    共现矩阵[ki][kj] += 1
    
共现窗口: 同一场景内（约2-5分钟的语义单元）
```

#### Step 2: 计算共现强度

```
共现强度(ki, kj) = 共现次数(ki, kj) / (出现次数(ki) + 出现次数(kj) - 共现次数(ki, kj))

这是 Jaccard 系数——衡量两个词的"绑定程度"
```

#### Step 3: 构建加权共现图

```
节点 = 关键词
边 = 共现强度（权重）
节点大小 = TF-W（初始权重）
```

#### Step 4: 计算网络中心性（三种互为补充）

```markdown
### 中心性1: 度中心性 (Degree Centrality)
DC(k) = 与k直接连接的关键词数 / 总关键词数
含义: 这个词和多少个不同的概念发生关联？

### 中心性2: 中介中心性 (Betweenness Centrality)  
BC(k) = 经过k的最短路径数 / 所有最短路径数
含义: 这个词是否是"概念桥梁"？去掉它，多少概念对会断开？

### 中心性3: 特征向量中心性 (Eigenvector Centrality)
EC(k) = 与k连接的词的重要性加权和
含义: 这个词不仅连接多，而且连接的都是重要的词（PageRank思想）
```

#### Step 5: 综合共现权重

```
CO-W(k) = α·DC(k) + β·BC(k) + γ·EC(k)

默认: α=0.3, β=0.4, γ=0.3
中介中心性权重最高——"概念桥梁"比"连接多"更有结构价值
```

### 输出格式

```markdown
### 共现权重 Top 15
| 排名 | 关键词 | 度中心性 | 中介中心性 | 特征向量中心性 | CO-W |
|------|--------|---------|----------|-------------|------|
| 1 | "价值交换" | 0.78 | 0.92 | 0.85 | 0.86 |
| 2 | "IP资产" | 0.72 | 0.81 | 0.90 | 0.81 |
| 3 | "信任前置" | 0.65 | 0.73 | 0.78 | 0.72 |
| ... | ... | ... | ... | ... | ... |

### 关键发现
- 高TF-W但低CO-W: [列出] → 高频但孤立的话题，可能是口头禅或重复案例
- 低TF-W但高CO-W: [列出] → 🔥 "隐藏枢纽"——出现不多但连接了很多重要概念
- 高中介中心性: [列出] → 概念桥梁——拆掉它们，整个体系散架
```

### 共现图可视化

```
        [信任前置]──0.8──[IP资产]──0.6──[复利思维]
            │                │               │
           0.5              0.7             0.4
            │                │               │
        [价值交换]────0.9───[长期主义]──0.3──[内容vs资产]
            │                                │
           0.6                             0.7
            │                                │
        [自知度]──────────────────────[不对称回应]
                    0.5

图例:
  节点大小 = TF-W（出现频率）
  边粗细 = 共现强度
  🔴 = 高中介中心性（概念桥梁）
  🔵 = 高特征向量中心性（被重要概念包围）
```

## Head 3: 位置权重 (POS-W, Positional Weight)

> **类比 Transformer**: 这相当于 Positional Encoding——同一个词出现在场景开头（立论位置）和出现在场景中间（举例位置），权重应该不同。

### 目的
关键词在场景中的位置决定了它的功能角色。

### 位置分段

```markdown
| 位置 | 占场景比例 | 功能角色 | 权重系数 |
|------|----------|---------|---------|
| **开场段** | 0-20% | 立论/定调/抛出核心判断 | ×1.5 |
| **展开段** | 20-70% | 论证/举例/分析 | ×1.0 |
| **高潮段** | 70-90% | 升华/最强烈的表达 | ×1.3 |
| **收尾段** | 90-100% | 总结/回扣/行动号召 | ×1.4 |
```

### 位置识别规则

```markdown
开场段识别:
  - 此人的第一段完整表达
  - 包含"首先""第一个""核心问题是"等信号词
  - 提问后的首次回应
  
高潮段识别:
  - 情绪强度峰值（来自Phase -1 情绪场）
  - 语速/音量突变
  - 包含"说白了""本质是""归根结底""最关键的"

收尾段识别:
  - 此人的最后一段完整表达
  - 包含"总结一下""所以我想说的是""记住这一条"
  - 情绪平静回归
```

### 计算方式

```
POS-W(k) = Σ(每个出现位置的位置系数 × 该位置的权重) / 总出现次数

其中每个出现位置的权重 = 该位置对应段的权重系数
如果同一个关键词在多个位置出现，取加权平均
```

### 输出格式

```markdown
### POS-W 特殊发现
| 关键词 | 总出现 | 开场占比 | 高潮占比 | 收尾占比 | POS-W | 角色判定 |
|--------|--------|---------|---------|---------|-------|---------|
| "IP资产" | 47 | 35% | 45% | 20% | 1.38 | 核心论题（高频+高开+高收） |
| "复利思维" | 38 | 10% | 60% | 30% | 1.32 | 论证工具（高频+偏展开+高高潮） |
| "信任前置" | 35 | 55% | 25% | 20% | 1.42 | 🔥 立论级概念（高频+偏高开） |
```

## Head 4: 跨场景一致性权重 (CSC-W, Cross-Scene Consistency Weight)

### 目的
衡量一个关键词是否稳定地出现在不同类型的场景中。

**直觉**: 
- 出现在多种场景类型中 → 跨场景核心概念 → 高权重
- 只出现在一种场景类型中 → 场景限定概念 → 降低通用性权重
- 分布越均匀 → 越可能是底层操作系统的一部分

### 计算方式

使用归一化熵（Normalized Entropy）衡量分布均匀度：

```
场景类型分布 = {场景类型: 出现次数}

例如 "IP资产" 的分布:
  对话问答: 15次
  单人讲述: 20次  
  实战诊断: 8次
  争论冲突: 2次
  闲谈互动: 2次
  
分布概率 P(k, 场景类型i) = 出现次数 / 总出现次数

熵 H(k) = -Σ P(k, i) × log(P(k, i))

归一化熵 = H(k) / log(场景类型总数)

CSC-W(k) = 归一化熵 × 跨场景因子

跨场景因子 = 出现场景类型数 / 总场景类型数
```

### 权重解读

```markdown
| CSC-W | 含义 | 蒸馏策略 |
|-------|------|---------|
| >0.8 | 🔵 跨场景核心概念——不论什么环境都出现 | 高优先级，可能是公理级概念 |
| 0.5-0.8 | 🟡 场景泛化概念——多个场景出现但有偏好 | 标注偏好场景 |
| 0.3-0.5 | 🟠 场景限定概念——主要在特定场景出现 | 标注适用场景范围 |
| <0.3 | 🔴 单场景特化——仅在一个场景类型中出现 | 不泛化，诚实边界标注 |
```

### 输出格式

```markdown
### CSC-W 分布分析
| 关键词 | CSC-W | 场景分布 | 判定 |
|--------|-------|---------|------|
| "价值交换" | 0.92 | 对话(15)/实战(12)/讲述(10)/闲谈(5)/冲突(3) | 🔵 公理级——任何场景都运行 |
| "不对称回应" | 0.75 | 对话(18)/实战(14)/冲突(8) | 🟡 互动场景偏好——在单人输出中不出现 |
| "流量打法" | 0.35 | 实战(20)/对话(5) | 🟠 诊断场景限定——方法论而非底层信念 |
```

## Head 5: 情绪强度权重 (EI-W, Emotional Intensity Weight)

### 目的
一个关键词在被谈论时的情绪强度反映了此人对它的**个人投入度**。

**直觉**: 
- 每次提到都情绪激动 → 这是此人的核心信念/痛点
- 提到时总是平静 → 可能是工具性概念
- 情绪变化大 → 此人对这个概念有不稳定的态度

### 计算方式

基于 Phase -1 的六维环境分析——情绪场数据：

```
每个关键词的每次出现都关联一个场景情绪强度(1-5)

EI-W(k) = 平均情绪强度 × 情绪方差系数

情绪方差系数:
  - 方差小（每次提及情绪一致）→ ×1.2（情绪稳定 = 态度稳定，是成熟概念）
  - 方差大（提及情绪波动）→ ×0.8（态度不稳定，可能是正在形成的概念）
  - 极端方差（有些场景激动有些场景冷漠）→ ×1.3（🔥 内在张力信号！）
```

### 输出格式

```markdown
### EI-W 情绪关联
| 关键词 | 平均情绪 | 方差 | EI-W | 解读 |
|--------|---------|------|------|------|
| "自知度" | 4.2 | 0.3 | 0.84 | 高度稳定高投入——此人每次讲都激动 |
| "内容资产化" | 3.8 | 0.8 | 0.61 | 中等投入有波动——可能在不同场景下态度不同 |
| "数据驱动决策" | 2.1 | 1.6 | 0.67 | 🔥 低投入+高波动——内在张力！在某些场景下他回避这个话题 |
| "流量" | 3.0 | 0.2 | 0.72 | 中等投入稳定——工具性概念，讨论时情绪平稳 |
```

## Head 6: 场景类型方差权重 (STV-W, Scene Type Variance Weight)

### 目的
这个概念在此人"做"的时候出现还是"说"的时候出现？

**直觉**:
- 在"实战诊断"（做）中高频出现 → 这是真正在用的概念
- 只在"单人讲述"（说）中出现 → 可能是修辞/包装
- 言行一致（说和做的场景都有）→ 最高权重

### 计算方式

```
将6种场景类型分为两大类:

"做"的场景: 实战诊断、对话问答、争论冲突
"说"的场景: 单人讲述、演讲展示、写作输出
"闲"的场景: 闲谈互动、复盘反思

言行比 = "做"场景中的出现次数 / "说"场景中的出现次数

STV-W 判定:
  言行比 ≥ 0.8 → "言行合一型" — STV-W × 1.5
  言行比 0.4-0.8 → "说多做少型" — STV-W × 0.8
  言行比 < 0.4 → "说而不做型" — STV-W × 0.5，⚠️ 标注"待验证"
  只有"做"场景 → "只做不说型" — STV-W × 1.3，🔥 可能是隐性方法论
  只有"说"场景 → "只说不做型" — STV-W × 0.4
```

### 输出格式

```markdown
### STV-W 言行对照
| 关键词 | "做"场景 | "说"场景 | 言行比 | STV-W | 判定 |
|--------|---------|---------|--------|-------|------|
| "不对称回应" | 22次 | 5次 | 4.4 | 1.3 | 🔥 只做不说——隐性方法论 |
| "IP资产化" | 8次 | 25次 | 0.32 | 0.5 | ⚠️ 说多做少——待验证是否真正执行 |
| "价值交换" | 18次 | 20次 | 0.9 | 1.5 | ✅ 言行合一——核心操作公理 |
```

## Head 7: 主动触发权重 (RT-W, Response Trigger Weight)

### 目的
这个概念是此人自己想说的，还是被别人问出来的？

**直觉**:
- 自己主动提出的 → 他的核心关注点 → 高权重
- 被问到才答的 → 他有关注但不会主动提 → 中等权重
- 多次被问但回避 → 🔥 可能是盲区或痛点

### 计算方式

基于 Phase -1 的触发条件数据：

```
每个关键词的每次出现，关联其所在场景的触发类型:
  - 主动输出: 此人主动发起这个话题
  - 借题发挥: 被问到但远超问题范围展开
  - 被动回应: 被问到就直接回答
  - 被迫回应: 被追问/质疑后才答

主动率 = (主动输出次数 + 借题发挥次数) / 总出现次数

RT-W(k) = 主动率 × 出现次数因子

出现次数因子 = min(1.0, 出现次数/10) — 对出现太少的词降低置信度
```

### 输出格式

```markdown
### RT-W 触发分析
| 关键词 | 主动% | 借题% | 被动% | 被迫% | RT-W | 解读 |
|--------|-------|-------|-------|-------|------|------|
| "IP资产" | 55% | 30% | 15% | 0% | 0.85 | 🔥 绝对主动——他自己想说 |
| "数据驱动" | 10% | 15% | 50% | 25% | 0.25 | 被动话题——被问才谈，且有时回避 |
| "失败案例" | 5% | 0% | 20% | 75% | 0.05 | 🔥🔥 回避信号——几乎从不主动提，被追问才回应 |
```

# Part 2: 多Head聚合

## 聚合公式

类似 Transformer 的 Multi-Head Attention 最后一步：

```
MultiHead(k) = Concat(TF-W, CO-W, POS-W, CSC-W, EI-W, STV-W, RT-W)

综合权重(k) = w1×TF-W + w2×CO-W + w3×POS-W + w4×CSC-W + w5×EI-W + w6×STV-W + w7×RT-W
```

### 默认权重配置

```markdown
| Head | 权重系数 | 理由 |
|------|---------|------|
| CO-W (共现) | 0.25 | 🔥 最像Transformer注意力的维度，权重最高 |
| CSC-W (跨场景一致性) | 0.20 | 区分"核心概念"vs"场景限定"的关键 |
| STV-W (言行比) | 0.15 | "做"比"说"重要 |
| TF-W (词频) | 0.10 | 基础信号，但容易误导（高频≠重要） |
| POS-W (位置) | 0.10 | 位置信息提供结构意义 |
| RT-W (主动触发) | 0.10 | 此人自己的优先级 |
| EI-W (情绪强度) | 0.10 | 情绪投入度 |
| 合计 | 1.00 | |
```

## 🆕 Step -1.5a: 自适应权重估计（v3.11新增 — 权重从数据涌现）

> **为什么需要**：v3.5的默认聚合系数（0.25/0.20/0.15/0.10...）是经验固化的，对大多数蒸馏场景够用。但当蒸馏对象的**数据结构非常特殊**时——比如全是短文（POS-W信号弱）、或全是对话（STV-W和RT-W信号强但TF-W易误导）、或素材类型极度不均衡——固定系数会导致次优权重分配。
>
> **核心理念**：不预设系数，而是根据**数据本身的特征**动态估计各Head的聚合权重。数据告诉我们应该更相信哪个维度。

### 自适应逻辑

```
输入: 场景文档全集 + 关键词列表
  ↓
Step A: 数据特征诊断 — 计算4个特征指标
  ↓
Step B: 基于特征动态调整7个Head的聚合系数
  ↓
Step C: 输出调整后的综合权重矩阵 + 调整理由
```

### Step B: 自适应系数计算

基于4个特征指标，动态调整7个Head的聚合系数：

```python
def adaptive_weights(F1_type_diversity, F2_env_diversity, F3_size, F4_alignment):
    """
    基于数据特征自适应计算7Head聚合系数。
    所有系数归一化到 sum=1.0。
    """
    # 基础系数（默认值作为先验）
    w = {
        'CO-W': 0.25,   # 共现
        'CSC-W': 0.20,  # 跨场景一致性
        'STV-W': 0.15,  # 言行比
        'TF-W': 0.10,   # 词频
        'POS-W': 0.10,  # 位置
        'RT-W': 0.10,   # 主动触发
        'EI-W': 0.10,   # 情绪强度
    }
    
    # 规则1: 场景环境多样性低 → CSC-W降权，CO-W升权
    if F2_env_diversity < 0.3:
        delta = min(0.10, w['CSC-W'] * 0.5)
        w['CSC-W'] -= delta
        w['CO-W'] += delta  # 用共现网络结构补充
    
    # 规则2: 素材类型单一（以对话为主）→ TF-W降权，STV-W升权
    if F1_type_diversity < 0.3:
        delta = min(0.05, w['TF-W'] * 0.5)
        w['TF-W'] -= delta
        w['STV-W'] += delta
    
    # 规则3: 言行一致性低 → STV-W/RT-W降权，CO-W/CSC-W升权
    if F4_alignment < 0.3:
        delta = min(0.08, w['STV-W'] * 0.3)
        w['STV-W'] -= delta / 2
        w['RT-W'] -= delta / 2
        w['CO-W'] += delta / 2
        w['CSC-W'] += delta / 2
    
    # 规则4: 小样本（场景<20）→ TF-W降权，CO-W升权（结构信息比频率信息更可靠）
    if F3_size == '小':
        delta = min(0.08, w['TF-W'] * 0.5)
        w['TF-W'] -= delta
        w['CO-W'] += delta / 2
        w['CSC-W'] += delta / 2
    
    # 规则5: 大样本（场景>100）→ TF-W升权，EI-W降权
    if F3_size == '大':
        delta = min(0.05, w['EI-W'] * 0.3)
        w['TF-W'] += delta
        w['EI-W'] -= delta
    
    # 规则6: 素材类型多样性极高 → POS-W升权（多种类型的结构信息互补）
    if F1_type_diversity > 0.7:
        delta = min(0.05, w['EI-W'] * 0.3)
        w['POS-W'] += delta
        w['EI-W'] -= delta
    
    # 归一化
    total = sum(w.values())
    return {k: round(v / total, 3) for k, v in w.items()}
```

### 使用原则

1. **默认优先**：如果数据特征诊断没有触发任何规则的极端条件（所有指标在中等范围），使用默认系数即可。
2. **人工确认**：自适应系数是建议，执行者确认合理后再应用。如果调整方向和蒸馏目标冲突（如蒸馏目标是构建AI分身但自适应建议降STV-W），以蒸馏目标为准。
3. **记录决策**：无论使用默认还是自适应，在权重报告中记录决策理由。
4. **可选步骤**：如果资源有限或数据量很小（<10个场景），可跳过自适应估计，直接使用默认系数。在诚实边界中注明"权重系数基于默认配置，未做数据自适应"。
