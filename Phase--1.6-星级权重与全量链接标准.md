# 星级权重体系 + 思维-案例链接矩阵 + 100%全量链接铁律

> **定位**：权重系统的"质量层"。Phase -1.5 计算出了数字权重，Phase -1.6 把这些权重转化为可对比的星级，并建立每个思维与案例之间的完整链接网络，强制要求100%内容互链。
>
> **核心问题**：
> 1. 一个思维值几颗星？——不是凭感觉，是基于层级/案例数/一致性/独特性 四维自动评定
> 2. 这个思维挂了多少案例？——每条链接都要量化，弱链接要预警
> 3. 有没有漏网之鱼？——100%的内容单元都必须被链接，一个都不能少

---

# Part 1: 星级权重体系（四维自动评定）

## 1.0 星级定义

| 星级 | 名称 | 含义 | 蒸馏价值 |
|------|------|------|---------|
| ⭐⭐⭐⭐⭐ | **公理级** | 去掉它，整个体系崩塌。跨场景/跨时间/跨情绪稳定运行。 | 构建AI分身的第一优先级 |
| ⭐⭐⭐⭐ | **元规则级** | 不局限于单一场景，决定了"何时用哪个操作级方法"。 | 理解此人决策逻辑的关键 |
| ⭐⭐⭐ | **操作级** | 有明确触发条件+操作步骤，但限于特定场景类型。 | 可复用的方法论 |
| ⭐⭐ | **行为倾向** | 重复出现但未达到方法论级别的统计标准。 | 了解偏好但不声称是"规则" |
| ⭐ | **待验证** | 案例不足或仅单场景出现，标注为观察中的模式。 | 诚实边界标注 |

## 1.1 四维评定标准

每颗星不是拍脑袋定的。四个维度各贡献一个子评分，加权合成最终星级。

### 维度1: 思维层级深度 (Thinking Depth, 权重40%)

基于涌现维度形式化步骤（Phase 2 Step 2.2）的 L0-L3 深度判定：

| 层级 | 分数 | 判断标准 |
|------|------|---------|
| **L3 公理级** | 5.0 | 去掉它就整个体系崩塌。不依赖其他模式，但其他模式依赖它。 |
| **L2 元规则级** | 4.0 | 跨场景稳定运行。决定"什么时候用操作层的哪个"。不局限于单一场景类型。|
| **L1 操作级** | 3.0 | 有明确触发条件+操作步骤+失效条件。但限于特定场景类型。 |
| **L0 行为倾向** | 2.0 | 重复≥3次但推导链不完整或可证伪条件不明确。 |
| **L- 待定** | 1.0 | 尚未完成形式化验证。 |

```
思维层级分数 = L级别对应分数 / 5.0（归一化到0-1）
```

### 维度2: 案例支撑数 (Case Support, 权重30%)

> **核心问题**：这个思维挂了多少个案例？挂钩数量直接决定可信度。

| 案例数 | 分数 | 判断 |
|--------|------|------|
| ≥20个案例 | 1.0 | 🟢 强支撑——可以说"此人确实是这样" |
| 10-19个案例 | 0.8 | 🟢 良好支撑 |
| 5-9个案例 | 0.6 | 🟡 基本支撑——可声称但加置信度标注 |
| 3-4个案例 | 0.4 | 🟠 弱支撑——标注"案例有限，暂定" |
| 1-2个案例 | 0.2 | 🔴 极弱支撑——标注"待更多案例验证" |
| 0个案例 | 0.0 | 🔴 无案例——不能声称，进诚实边界 |

**案例计数规则**：
```
一个案例 = 一个不同的场景 / 一个不同的素材来源
同一素材中的不同段落 = 如果触发条件不同 → 算不同案例
此人自己引用的案例 = 算案例，但标注"此人自述"（降权×0.7）
二手来源的案例 = 算案例，但标注"二手"（降权×0.5）
```

### 维度3: 跨场景一致性 (Cross-Scene Consistency, 权重20%)

基于 Phase -1.5 的 CSC-W：

| CSC-W 范围 | 分数 | 判断 |
|-----------|------|------|
| >0.8 | 1.0 | 几乎在所有类型的场景中都出现 |
| 0.6-0.8 | 0.8 | 出现在多数场景类型中 |
| 0.4-0.6 | 0.6 | 出现在部分场景类型中 |
| 0.2-0.4 | 0.4 | 仅1-2种场景类型中出现 |
| <0.2 | 0.2 | 仅单一场景类型，可能是巧合 |

### 维度4: 人物独特性 (Uniqueness, 权重10%)

基于 Phase 3 对照定位结果：

| 判定 | 分数 | 判断 |
|------|------|------|
| 此人独有（14人物库中无人有此模式） | 1.0 | 🔥 定义性的——是他之所以是他的关键 |
| 与此类型的典型人物相似但变异 | 0.7 | 有特色——是类型内的独特变异 |
| 此类型的典型特征，此人标准表现 | 0.5 | 类型共有——有价值但不定义独特性 |
| 通用特征，几乎所有人都有 | 0.3 | 基础特征——必要但不突出 |

### 维度5: 时效性 (Time Sensitivity, 权重扣减项)（🆕v3.9 修复C4）

> **问题**：原公式不关心时间。一个方法可能是此人"早期用的，后来放弃了"，但如果早期案例多，它反而被评为高星级→过时方法论被推荐给学习者。
> **方案**：时效性不是加分项，而是**扣减项**——最近未出现的方法自动降权。

| 最近一次出现 | 扣减系数 | 说明 |
|------------|---------|------|
| 最近3个月内 | ×1.00 | 活跃模式，无扣减 |
| 3-6个月前 | ×0.90 | 近期减少，轻微扣减 |
| 6-12个月前 | ×0.75 | 可能正在被放弃，显著扣减 |
| 12-24个月前 | ×0.50 | 大概率已过时，半星扣减 |
| >24个月前或从未出现 | ×0.30 | 几乎可以判定为"历史模式"，降一星 |

**时效性判定规则**：
```
1. 如果有明确的时间线证据（Step 1.3产出）表明此人已明确放弃此方法论 → ×0（不参与星级评定，进入"历史模式库"）
2. 如果素材时间跨度<6个月 → 时效性默认为×1.00（素材不够长，无法判断时效性），标注诚实边界
3. "最近出现"以SRC-ID的时间戳为准
```

**时效性在输出中的标注**：
```markdown
| 思维/概念 | 星级(无时效) | 最近出现 | 时效扣减 | 最终星级 |
|----------|------------|---------|---------|---------|
| "价值交换" | ⭐⭐⭐⭐⭐ | 2024-06 (2周前) | ×1.00 | ⭐⭐⭐⭐⭐ |
| "旧方法X" | ⭐⭐⭐⭐ | 2022-03 (2年前) | ×0.30 | ⭐⭐⭐ |
| "早期模式Y" | ⭐⭐⭐ | 此人已明确放弃 | ×0 | 历史模式(不进正式评级) |
```

## 1.2 综合星级计算（v3.9修订）

```
综合得分 = (0.40 × 思维层级 + 0.30 × 案例支撑 + 0.20 × 一致性 + 0.10 × 独特性) × 时效性扣减

星级映射:
  综合得分 ≥ 0.90 → ⭐⭐⭐⭐⭐ (公理级)
  综合得分 ≥ 0.70 → ⭐⭐⭐⭐ (元规则级)
  综合得分 ≥ 0.50 → ⭐⭐⭐ (操作级)
  综合得分 ≥ 0.30 → ⭐⭐ (行为倾向)
  综合得分 < 0.30 → ⭐ (待验证)
  
  时效性扣减后星级最多降1级（如⭐⭐⭐⭐→⭐⭐⭐），除非此人已明确放弃（→历史模式库）
```

### Phase -1.5权重与星级的关系（🆕v3.9 修复A4）

> **问题**：Phase -1.5产出"数据权重"（如CO-W=0.83），Phase -1.6产出"星级"（如⭐⭐⭐⭐）。两个数字可能不一致——以谁为准？
> **答案**：两者回答不同的问题，不冲突，但需明确各自用途。

| | Phase -1.5 数据权重 | Phase -1.6 星级 |
|---|-------------------|----------------|
| **回答的问题** | "这个关键词/概念在此人的素材中客观上有多重要？" | "这个方法论/思维有多成熟、多可靠？" |
| **计算基础** | 7维统计数据（词频/共现/位置/一致性/情绪/言行/触发） | 4+1维质量评定（层级/案例/一致性/独特性+时效） |
| **驱动因素** | 数据驱动的——从素材统计中涌现 | 质量驱动的——从方法论完整性中评定 |
| **用途** | 决定维度激活优先级、关键词排序、概念权重 | 决定方法论成熟度、Skill优先级、产物中强调程度 |
| **冲突处理** | 如果数据权重高但星级低 → 说明"此人大量谈论X，但X的方法论不够成熟/证据不足" → 标注为"高频但未成熟" | |
| | 如果数据权重低但星级高 → 说明"此人很少谈Y，但Y的方法论很成熟" → 标注为"低频但高质量" | |
| | **以星级为准做产物质量判断，以数据权重做内容优先级排序。两者不打架——是分工。** | |

```
示例（鲁大魔）:
- "价值交换": 数据权重0.92, 星级⭐⭐⭐⭐⭐ → 🟢 高频+高质量 → Skill第一优先级
- "归因翻转": 数据权重0.76, 星级⭐⭐⭐ → 🟡 中频+中等质量 → 可做Skill但标注置信度
- "沉默开场": 数据权重0.45, 星级⭐⭐ → 🟡 低频+低质量 → 仅作参考
- "旧方法X": 数据权重0.65, 星级(无时效)⭐⭐⭐⭐, 但时效×0.30 → 🔴 曾经高频但已过时 → 标注为"历史方法"
```

# Part 2: 思维-案例链接矩阵

## 2.0 核心概念

```
思维节点 (CON / OPI / PAT) ←→ 案例节点 (CAS) ←→ 场景节点 (SCN) ←→ 素材节点 (SRC)

          链接线 = 每个思维与案例之间的引用关系
          链接强度 = 案例出现的次数 + 场景类型多样性 + 原文引用完整度
```

## 2.1 链接计数规则

### 每个思维方式（PAT/CON）挂了多少案例？

```markdown
| 思维方式 | 直接案例数 | 间接案例数 | 总链接数 | 强链接(★★★) | 中链接(★★☆) | 弱链接(★☆☆) | 链接密度 |
|---------|----------|----------|---------|------------|------------|------------|---------|
| "不对称回应" | 28 | 12 | 40 | 22 | 13 | 5 | 1.43/场景 |
| "价值交换" | 23 | 15 | 38 | 18 | 14 | 6 | 1.52/场景 |
| "归因翻转" | 8 | 4 | 12 | 5 | 5 | 2 | 0.48/场景 |
```

### 计数定义

```
直接案例: 此案例的核心分析直接服务于这个思维方式
  例: CAS-001 "餐饮老板诊断" → PAT-001 "不对称回应"（此案例的核心就是展示不对称回应）

间接案例: 此案例提到了这个思维方式，但不是案例的核心
  例: CAS-005 "知识付费讨论" → 在讨论中顺带展示了PAT-001（但不是该案例的主体）

强链接(★★★): 原文中此人明确表达/展示了该思维方式 + 有完整触发→行为→结果链
中链接(★★☆): 原文中可观察到该思维方式但不完整
弱链接(★☆☆): 分析者推断此人此处用了该思维方式，但原文不够明确

链接密度 = 总链接数 / 出现场景数
  高密度(>1.0): 此思维方式在每个出现场景中被多维度展示
  低密度(<0.5): 此思维方式的展示比较单薄
```

## 2.2 反向链接：每个案例挂了多少思维方式？

```markdown
| 案例 (CAS) | 场景 (SCN) | 链接的思维方式 | 链接数 | 核心启发 | 
|-----------|-----------|-------------|--------|---------|
| CAS-001 "餐饮老板诊断" | SCN-001-01 | PAT-001(★★★), PAT-003(★★★), PAT-005(★★☆), CON-002(★★☆) | 4 | 演示了诊断引擎的完整弧线 |
| CAS-002 "知识付费案例" | SCN-001-03 | PAT-002(★★★), CON-001(★★☆), PAT-004(★☆☆) | 3 | 展示了价值交换在具体场景的应用 |
| CAS-003 "团队管理讨论" | SCN-002-01 | PAT-001(★☆☆) | 1 | 🔴 链接弱——此案例的蒸馏价值可能未被充分挖掘 |
```

### 关键检测

```markdown
### 链接异常检测
| 异常类型 | 检测条件 | 发现 | 行动 |
|---------|---------|------|------|
| **孤岛案例** | CAS链接数=0 或=1且弱链接 | CAS-015 "闲聊片段"仅弱链接PAT-007 | 返回重分析或进诚实边界 |
| **过载思维** | PAT链接数远超均值(>2σ) | PAT-001有40条链接(均值=12) | ✅ 正常——这是核心公理应有的链接密度 |
| **欠载思维** | PAT链接数<3且星级≥⭐⭐⭐ | — | 🔴 矛盾——高星级但低链接，检查星级是否虚高 |
| **单向案例** | CAS→PAT有链接但PAT↛CAS | CAS-008被PAT-003引用但CAS-008的frontmatter中未标注 | 🔴 双向性断裂——补链 |
| **死链接** | 引用了不存在的ID | PAT-005引用CAS-099(不存在) | 🚫 硬性阻断——修复 |
```

## 2.4 链接密度分析

```markdown
### 网络统计
- 总思维节点: N
- 总案例节点: M
- 总链接线: L
- 平均每个思维链接案例数: L/N
- 平均每个案例被思维方式数: L/M
- 网络密度: 2L / (N × (N+M-1)) = X%

### 密度判定
| 密度 | 判定 | 解读 |
|------|------|------|
| >15% | 🟢 稠密 | 知识网络发达，概念-案例相互支撑充分 |
| 8-15% | 🟡 正常 | 大部分概念有案例支撑 |
| 3-8% | 🟠 稀疏 | 较多概念缺乏案例支撑 |
| <3% | 🔴 极疏 | 大量孤岛，蒸馏质量存疑 |

### 关键链接路径
找出最长的强链接链：
PAT-001(价值交换)─★★★─CAS-001─★★★─PAT-003(不对称回应)─★★★─CAS-004─★★★─CON-002(信任前置)
→ 这条链展示了此人的完整思维链路：从公理到操作到案例到概念
```

# Part 3: 100%全量链接铁律

## 3.0 总则

```
铁律: 蒸馏完成时，100%的核心内容单元必须被链接。100%的核心素材必须被引用。
这不是"尽力而为"——是硬性门禁。核心素材遗漏=产物不可发布。
```

> ⚠️ **v3.8 修复C3**：原版要求"素材覆盖率=100%"可能导致大量弱链接被强造出来充数（场景过渡、寒暄、无关闲聊也必须链接）。改为"核心素材覆盖率=100%"，允许标注哪些素材是"背景素材"不参与链接计算。

### 3.0.1 核心素材 vs 背景素材（v3.8新增）

在素材注册阶段（Step 0.0），对每条素材标注其**链接等级**：

| 链接等级 | 定义 | 示例 | 链接要求 |
|---------|------|------|---------|
| **🔴 核心素材** | 包含可提取的方法论、观点、概念、案例判断 | 此人阐述核心方法论、做关键决策、表达独特观点 | **必须100%链接** |
| **🟡 支撑素材** | 为核心素材提供上下文、补充细节 | 展开说明、举例、回应追问 | 至少被1个核心素材间接引用 |
| **⚪ 背景素材** | 场景过渡、寒暄、重复内容、纯信息传递 | "好的""下一个问题""今天天气不错" | 不要求链接，但需标注为背景 |

**判定标准**：
- 如果一条素材中没有任何CON/OPI/CAS/PAT可提取 → 标记为⚪背景
- 如果一条素材中至少有一个可提取的认知单元 → 标记为🔴核心
- 如果一条素材只是补充/展开已有的核心素材 → 标记为🟡支撑

**背景素材比例上限**：背景素材占总素材数 ≤15%。超过15%→检查是否误标。

## 3.1 100%的定义（v3.8修订）

| 检查项 | 100%标准 | 验证方式 |
|--------|---------|---------|
| **核心素材覆盖率** | 100%的🔴核心SRC-ID在至少1个内容单元的 source_documents 中出现 | 全量扫描, 0容忍 |
| **支撑素材覆盖率** | ≥80%的🟡支撑SRC-ID被间接引用 | 全量扫描 |
| **背景素材覆盖率** | 不要求，但⚪占比须≤15% | 比例检查 |
| **案例覆盖率** | 100%的CAS被至少1个PAT/CON/OPI引用 | 反向检查每个CAS |
| **观点覆盖率** | 100%的OPI在关系网络中被至少1个其他单元引用 | 孤儿检测 |
| **概念覆盖率** | 100%的CON在关系网络中被至少1个其他单元引用 | 孤儿检测 |
| **场景加工率** | 100%的SCN产出了至少1个内容单元(CON/OPI/CAS) | 场景→单元映射检查 |
| **双向链接率** | 100%的关系是双向的（A引用B→B的frontmatter也标注被A引用） | 双向性全量检查 |
| **死链数** | 0（零容忍） | 全量[[引用]]解析 |

## 3.3 例外处理（唯一可接受的未链接情况）

```
以下情况允许内容单元未被100%链接，但必须进入诚实边界并标注原因：

1. 🟡 新发现但未整合: Phase 4交叉分析中刚发现的新模式，尚未建立全部链接
   → 标注为"待整合" + 给定期限（下一版本蒸馏必须整合）

2. 🟡 素材质量限制: 单次提及且无法验证（如"此人提到过一句X但再无其他证据"）
   → 标注为"单次提及，证据不足" + 降权

3. 🔴 分析者推断: 分析者的推测而非直接证据
   → 标注"推断" + 推测依据 + 不进正式库

任何其他原因的不链接 → 不允许。
```

## 3.4 100%验证流程

```
Phase 4 结束后、Phase 5 产物生成前：

Step 验证-1: 素材覆盖率检查
  tool: coverage_scanner.py
  标准: 100%的SRC-ID被引用
  不通过: 返回遗漏的素材重新分析

Step 验证-2: 死链全量扫描
  tool: link_checker.py
  标准: 死链数 = 0
  不通过: 修复所有死链，重新扫描直到=0

Step 验证-3: 双向链接验证
  tool: relationship_analyzer.py
  标准: 100%的关系是双向的
  不通过: 补全缺失方向

Step 验证-4: 孤儿单元检测
  tool: relationship_analyzer.py --orphan-check
  标准: 孤立单元=0（或全部有豁免标注）
  不通过: 为孤岛建立链接

Step 验证-5: 链接强度扫描
  tool: relationship_analyzer.py --strength-check
  标准: 无★☆☆链接的高星级(≥⭐⭐⭐)单元
  不通过: 检查星级是否虚高

全部5项通过 → 🟢 门禁通过 → 进入Phase 5
任1项不通过 → 🚫 阻断 → 返回修复
```
