# v3.0 溯源铁律嵌入标准

> **目的**：把溯源从"贴在墙上的口号"变成"嵌入每个提示词的引擎"。
> **覆盖缺口**：#1 溯源未嵌入执行层、#2 原文保存格式、#3 引用颗粒度分级、#4 转述退化链追踪、#5 素材完整性声明

---

## Part 1: 原文保存格式标准（补缺口#2）

### 1.1 每种素材类型的保存格式

| 素材类型 | 文件格式 | 命名规则 | 时间戳精度 | 必需元数据 |
|---------|---------|---------|----------|----------|
| **视频→文字** | `.srt` + `.md` (全文transcript) | `SRC-XX-001_transcript.md` | MM:SS | 日期、时长、标题、来源URL、参与者 |
| **音频→文字** | `.md` (全文transcript) | `SRC-XX-001_transcript.md` | MM:SS | 日期、时长、标题、参与者 |
| **书籍/PDF** | `.md` (章节摘录) 或 原始PDF | `SRC-XX-001_ch3.md` | 章.节.段 | 书名、版本、ISBN、页码 |
| **社交媒体** | `.json` (导出) + `.md` (精选) | `SRC-XX-001_tweets.json` | 精确到分钟 | 平台、账号、导出日期、总条数 |
| **网页/博客** | `.md` (全文) | `SRC-XX-001.md` | 不适用 | URL、抓取日期、作者 |
| **直播录屏** | `.srt` + `.md` | `SRC-XX-001_transcript.md` | MM:SS | 日期、时长、平台、参与者 |
| **演讲/Keynote** | `.srt` + `.md` + 截图(可选) | `SRC-XX-001_transcript.md` | MM:SS | 日期、场合、听众 |
| **内部文档** | `.md` (扫描件则保留原件+转文字) | `SRC-XX-001.md` | 文档日期 | 来源、密级、获取方式 |

### 1.2 transcript 标准格式

```markdown
---
src_id: SRC-LDM-001
type: 直播连麦
date: 2024-03-15
duration: 45:32
platform: 抖音
participants: [鲁大魔, 连麦用户-网名]
source_url: https://...
transcriber: [自动转写工具名称]
quality: 原始转写/人工校对
---

# SRC-LDM-001: [标题]

[00:00] [说话人A]: [原话]
[00:15] [说话人B]: [原话]
[01:22] [说话人A]: [原话]
...

## 关键节点索引
- 03:45 — 鲁大魔开始破锁
- 12:30 — 连麦用户自述执行结果
- 28:00 — 鲁大魔给具体行动路径
- 42:00 — 收尾总结
```

### 1.3 素材文件目录结构

```
01-原始素材区/
├── _注册表.md                         # 素材ID总注册表
├── SRC-XX-001/                        # 每条素材一个文件夹
│   ├── original/                      # 原始文件（不修改）
│   │   ├── video.mp4 (或访问路径)
│   │   └── auto_transcript.srt
│   ├── transcript.md                  # 标准化的transcript
│   └── metadata.yaml                  # 元数据
├── SRC-XX-002/
│   └── ...
└── _完整性声明.md                     # 素材完整性声明
```

## Part 2: 三级引用颗粒度标准（补缺口#3）

### 2.1 颗粒度定义

| 级别 | 格式 | 可追踪到的位置 | 何时使用 |
|------|------|-------------|---------|
| **L1: 素材级** | `SRC-XX-003` | 整条素材 | 引用整段素材的背景/语境/氛围 |
| **L2: 位置级** | `SRC-XX-003, 12:34-12:58` | 素材中的具体段落 | 引用具体观点、故事、分析 |
| **L3: 精确级** | `SRC-XX-003, 12:34, "[此人原话]"` | 素材中的具体句子 | 引用关键定义、概念、转折点 |

### 2.2 各内容单元的最低引用级别

| 内容单元 | 最低级别 | 建议级别 | 例外说明 |
|---------|---------|---------|---------|
| **CON（概念定义）** | L2 | L3 | 如果此人定义来自多个来源，至少1个L3 |
| **OPI（观点）** | L2 | L3 | 轻量级OPI可接受L2 |
| **CAS（案例）** | L2 | L2 | 如果此人多次引用同一案例，至少1个L2 |
| **SOL（方案）** | L2 | L3 | 可操作步骤如果只是分析者推断，标注L2+推测 |
| **核心信念声明** | L3 | L3 | 必须有精确引用，不接受L2 |
| **表达DNA特征** | L2 | L3 | 高频词统计需要全量统计（不是样本） |
| **内在张力** | L3 | L3 | 矛盾的双方都必须有精确引用 |
| **底层运算规则** | L2 | L3 | 每个验证案例至少L2 |

### 2.3 升级和降级规则

```
降级规则（从L3→L2→L1）:
- 找不到精确原话但记得大概内容 → L2 + 标注"原文待核实"
- 只能找到此人的类似表达但不完全一致 → L2 + 标注"近似引用"
- 多处素材都有类似表达但无一处可精确引用 → L1 + 标注"多来源综合"

升级规则:
- 如果新素材提供了更精确的引用 → 升级引用级别
- 如果审查发现之前标记为L3的引用有误 → 降级+写入纠错日志
```

## Part 3: 转述退化链追踪（补缺口#4）

### 3.1 转述距离定义

```
L0: 原文直接引用
    格式: "[此人原话，一字不改]" — SRC-XX-003, 12:34
    标志: 引号内是此人的原词原句

L1: 一次转述
    格式: 此人认为 [X]（基于: "[原文]" — SRC-XX-003, 12:34）
    标志: 用自己的话概括，但附了原文对照

L2: 二次抽象
    格式: 此人的核心方法论可以概括为 [Y]（基于L1概括，原文见 SRC-XX-003, 12:34-15:00）
    标志: 从L1概括中进一步抽象，标注了"基于L1概括"

L3: 综合推断
    格式: 综合 SRC-XX-003, SRC-XX-007, SRC-XX-015 中此人的表达，推断其底层逻辑为 [Z]
    标志: 多个来源的综合判断，标注了"推断"
```

### 3.2 转述距离与可接受度

| 产物层次 | 最大可接受距离 | 例外 |
|---------|-------------|------|
| Skill的"角色扮演规则" | L1 | 语音特征等只能是L1（原文转述） |
| Skill的"心智模型" | L2 | 严禁L3——L3必须进入诚实边界 |
| Skill的"决策启发式" | L2 | 每条启发式必须至少1个L0原文对照 |
| Skill的"表达DNA" | L1 | DNA的例句必须是L0 |
| 知识库的CON | L1 | CON的定义必须有L0原文支撑 |
| 知识库的OPI | L1 | OPI是此人的观点，必须用L0原文 |

### 3.3 退化追踪标记

每个内容单元在YAML frontmatter中新增 `abstraction_level` 字段：

```yaml
---
id: CON-001
abstraction_level: L1
abstraction_chain:
  - level: L0
    source: SRC-XX-003, 12:34
    quote: "[此人原话]"
  - level: L1
    summarized_as: "[分析者的概括]"
    reviewer: [谁做的这个概括]
    date: YYYY-MM-DD
---
```

## Part 4: 溯源引擎嵌入每个维度提示词（补缺口#1 — 核心修复）

### 4.1 原则

v2.0的问题：溯源铁律在Phase开头声明一次，但每个维度提示词内部没有溯源步骤。

v3.0的修复：**每个维度提示词的输出格式中，溯源不是"质量检查"的可选项——是输出模板的必填项。**

### 4.2 嵌入模板

每个Phase维度的提示词，在输出格式中必须包含以下溯源区块：

```markdown
## [维度名] — [人物名]

[维度分析内容...]

### 📋 溯源报告（必填，不可跳过）
| 声明类型 | 声明内容 | 引用级别 | 溯源 |
|---------|---------|---------|------|
| 核心定义 | [维度的核心定义] | L2+ | SRC-XX-XXX, [位置] |
| 关键证据 | [支撑核心判断的证据] | L2+ | SRC-XX-XXX, [位置] |
| 统计数据 | [频率/百分比等] | L1 | 基于 SRC-XX-001 至 SRC-XX-050 的全文统计 |
| 推测 | [任何不是直接观察到而是推断的] | — | 推测依据: [X] — 可信度: [高/中/低] |

### ⚠️ 溯源警告
- 以下声明无法溯源到原文: [列出]
- 以下声明基于推测（非直接观察）: [列出]
```

### 4.3 逐维度溯源要求速查表

| Phase | 维度 | 最低溯源要求 |
|-------|------|------------|
| **1.1** | 表达DNA | 每条句式特征≥10条例句(带SRC-ID)；高频词附全量统计；口头禅附频率(N次/万字) |
| **1.2** | 智识谱系 | 每个明确提及来源附引用次数；每个推测影响附推测依据+可信度+独立交叉验证 |
| **1.3** | 时间线 | 每个转折点附前后原文；核心信念附跨年度引用(≥2年跨度) |
| **1.4** | 他者视角 | 每条评价附来源(谁/在哪/什么时候)；合理性评估附评判逻辑 |
| **1.5** | 内在张力 | 矛盾双方各附原文；标注双方出现的频率对比 |
| **1.6** | 概念体系 | 每个CON有L2+原文出处；典型应用≥5个带SRC-ID；失效条件附来源 |
| **1.7** | 观点库 | 每个OPI有L2+原文引用；质量分层附理由；多来源OPI标注所有来源 |
| **1.8** | 案例库 | 每个CAS有此人的原文分析；重复引用的案例标注各次出现的SRC-ID |
| **1.9a** | 关系网络 | 每条关系标注建立依据（SRC-ID中此人明确说了什么，或分析者根据什么推断） |
| **1.10** | 身份卡 | 每句自述可追溯到具体的素材或综合判断（标注哪些是综合） |
| **1.11** | 诚实边界 | 每条局限引用Phase 0的具体数据；提升路径具体到"补X条Y类型的素材" |
| **2.1** | 核心操作方法 | 每个方法≥3验证案例(带SRC-ID)；≥2反例；推导链的每个前提标注来源 |
| **2.2** | 底层运算规则 | 每个规则≥5验证案例；≥2反例；推导链每个环节可独立验证 |
| **2.3** | 价值观与反模式 | 每条价值观附冲突证据(带SRC-ID)；反模式附此人的明确反对原文 |
| **2.4** | 决策启发式 | 每条启发式≥2验证案例；适用边界+失效条件附来源 |
| **3.x** | 特有维度 | 溯源标准同Phase 2 — ≥3验证案例+≥2反例 |

## Part 6: 全文强制溯源检查清单（给执行者用）

在每次蒸馏开始前和执行中，按此清单逐项检查：

```
Phase 0 开始前:
[ ] 所有素材有唯一SRC-ID
[ ] 所有素材有标准化transcript（如适用）
[ ] 已输出"素材完整性声明"

Phase 1-3 每个维度完成后:
[ ] 溯源报告已填写（不可跳过）
[ ] 所有核心声明可追溯到SRC-ID
[ ] 推测性内容已标注"推测"+可信度
[ ] 无法溯源的内容已进入该维度的"溯源警告"

Phase 4 交叉分析完成后:
[ ] 交叉分析中的新结论有溯源（回溯到具体的Phase维度报告）
[ ] 交叉分析发现的矛盾已标注双方溯源级别

Phase 5 产物生成前:
[ ] 产物的溯源覆盖率≥90%
[ ] 产物的转述距离≤L2
[ ] 无法溯源的内容已在诚实边界中声明

Phase 5 产物生成后:
[ ] Step 5.3双链终检通过（全量死链检查=0）
[ ] 随机抽查N条结论可反查到原文
```

## 一句话总结

**v2.0的溯源是"声明"——v3.0的溯源是"引擎"。每个维度输出时必须有溯源报告，每个声明必须有引用级别，每个推测必须有可信度标注。不是贴在墙上的口号，是每个提示词输出模板里的必填项。**
