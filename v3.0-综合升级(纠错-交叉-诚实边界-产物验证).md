# Part A: 执行中的纠错与回退机制（补缺口#17）

## A.1 问题定义

v2.0的流水线是严格线性的——Phase 0→1→2→3→4→5。但实际蒸馏中，Phase 3的发现可能推翻Phase 1的结论。没有纠错回退机制，错误会一直带到产物中。

## A.2 纠错触发条件

| 触发类型 | 具体条件 | 示例 |
|---------|---------|------|
| **下游反推上游** | Phase N的结论与Phase N-1冲突 | Phase 3发现此人某次诊断用了"直接给答案"，但Phase 2提取的规则是"从不直接给答案" |
| **新素材推翻旧结论** | 动态更新中，新素材暴露了之前未见的模式 | 新素材中此人明确说"我以前那个方法错了" |
| **交叉分析暴露矛盾** | Phase 4发现两个维度的结论逻辑上不可共存 | 表达DNA说是"先结论后铺垫"，但时间线分析发现早期是"先铺垫后结论" |
| **产物验证失败** | 盲测中发现蒸馏产物给出了不符合此人的回答 | 蒸馏产物对某类问题给出的回答与此人的真实回答差异显著 |

## A.3 回退流程

```
Step B1: 标注冲突
  - 冲突点: [Phase X的结论A] vs [Phase Y的发现B]
  - 冲突类型: 逻辑矛盾 / 事实矛盾 / 新证据推翻旧推断
  - 严重程度: 🔴 阻断级（下游多个单元依赖此结论）/ 🟡 修正级 / 🟢 标注级

Step B2: 影响范围评估
  - 直接依赖被推翻结论的下游单元: [列表]
  - 间接依赖的单元: [列表]
  - 如果修改此结论，总受影响单元数: N

Step B3: 回退决策
  - 🔴 阻断级 → 立即回退到最早受影响的Phase，重新执行
  - 🟡 修正级 → 在当前Phase完成后回退，修正受影响单元
  - 🟢 标注级 → 不中断流程，标注"此结论存在冲突，见纠错日志"

Step B4: 执行回退
  - 回退到受影响的最早Phase
  - 修改受影响单元
  - 向下游传播修改（如果CON被修改，所有引用此CON的OPI重新检查）
  - 版本号递增
  - 写入纠错日志

Step B5: 纠错日志（继承dbs-correction格式）
  ```yaml
  ---
  id: CORR-001
  date: YYYY-MM-DD
  severity: 🔴阻断
  conflict: "[Phase 3 发现] vs [Phase 1 结论]"
  affected_units: [CON-003, OPI-007, OPI-012]
  resolution: "[如何修正]"
  lessons: "[这次纠错揭示的蒸馏方法改进]"
  ---
  ```
```

## A.4 纠错日志的持续价值

纠错日志不是问责记录——是方法论改进引擎。每次纠错后分析：
- 这次纠错是因为什么样的推理错误？
- 框架的哪个步骤可以改进以避免类似错误？
- 这个纠错类型是否在多个蒸馏中都出现了？

# Part B: Phase 4 交叉分析升级（补缺口#18）

## B.1 v2.0 → v3.0 变化

| v2.0 Phase 4 | v3.0 Phase 4 |
|-------------|-------------|
| 6个两两交叉 | 6个两两交叉 + 3个三重交叉 + 全维度综合评分 |
| 仅分析 | 分析 + 发现问题 → 触发纠错回退 |
| 输出报告 | 输出报告 + 更新诚实边界 |
| 无评分 | 自洽度/一致度/置信度/通用度 四维评分 |

## B.2 新增: 三重交叉

### 三重交叉1: 概念 × 时间线 × 智识谱系

```
目的: 画出思想演进的三维地图——每个概念来自哪、什么时候出现、怎么演变。

分析:
1. 对每个CON，标注: 出现时间 + 智识来源 + 此后是否变化
2. 找出"概念链"——CON-A → CON-B → CON-C（一个概念催生了下一个）
3. 找出"概念终结点"——哪些概念不再使用（末次出现后≥2年未再提）
4. 标注"借来的概念"（有明确智识来源）vs "原创概念"（无明确来源）

输出:
- 概念生命线图（按时间排列的概念出现/消失/演变）
- 智识来源时间线（什么时候接触了哪个影响源→产生了哪个概念）
- 此人何时停止了向外部学习？何时开始主要从自己的经验中提炼？
```

### 三重交叉2: 表达DNA × 内在张力 × 他者视角

```
目的: 语言特征在矛盾时如何变化？外部是否注意到了这些矛盾？

分析:
1. 当此人讨论矛盾涉及的双方主题时，表达DNA是否有变化？
   - 讨论主张A时: 短句/确定/不引用
   - 讨论主张B时: 长句/犹豫/大量引用
   → 揭示他更自信于哪一方
2. 他者视角是否指出了同样的内在矛盾？
   - 被指出的矛盾 → 可能更真实（外部也观察到了）
   - 未被指出的矛盾 → 可能更隐蔽（只有深度分析能发现）
3. 他是否在回应批评时改变了表达方式？

输出:
- 表达DNA随主题切换的变化图
- 外部批评与内在张力的重合度
```

### 三重交叉3: 操作方法 × 底层规则 × 价值观

```
目的: 操作→规则→价值 的层级推导是否自洽？

分析:
1. 每个操作方法可以从底层规则推导出来吗？
2. 每个底层规则可以从价值观推导出来吗？
3. 有没有"孤独的操作方法"（有方法但找不到支撑的规则和价值观）→ 可能是"伪方法（只是偶然行为）"
4. 有没有"断裂的推导链"（规则说应该做A，价值观暗示应该做B，但操作方法做的是C）→ 标注为深层矛盾

输出:
- 操作-规则-价值 完整推导树
- 孤独方法列表（可能是伪方法）
- 推导断裂列表（可能是真正的深层矛盾）
```

## B.3 新增: 全维度综合评分

```markdown
## 蒸馏质量综合评分 — [人物名]

### 思想体系自洽度: [X%]
计算方式: 1 - (内在矛盾数 / 总观点数)
- ≥90%: 高度自洽 → 此人的思想体系内部一致性强
- 70-90%: 有矛盾但可共存
- <70%: 存在显著矛盾 → 诚实边界必须大幅说明

### 言行一致度: [X%]（有决策记录时）
计算方式: 决策记录中符合声称原则的比例
- ≥80%: 言行高度一致
- 50-80%: 有差距
- <50%: 显著言行不一 → 内在张力的A类矛盾
- 无法计算: 缺少决策记录 → 诚实边界

### 蒸馏置信度: [X%]
综合一手占比(权重40%) + 时间跨度(权重30%) + 覆盖面(权重20%) + 交叉验证强度(权重10%)
- ≥80%: 高置信度
- 60-80%: 中置信度
- <60%: 低置信度 → 诚实边界必须占显著篇幅

### 方法论通用度: [L0/L1/L2/L3]
- 基于跨人物验证的最高级别
- 标注每个级别的占比（L0:N个, L1:N个, L2:N个, L3:N个）

### 蒸馏完整度: [X%]
已完成维度数 / 总激活维度数
- ≥90%: 完整
- 70-90%: 大部分完整
- <70%: 不完整 → 诚实边界列出未执行维度
```

# Part C: 活诚实边界（补缺口#19）

## C.1 核心变更

v2.0的诚实边界是"一次生成"的（Step 1.11做一次，Step 4.2汇总）。v3.0改为"持续更新的活文档"。

## C.2 诚实边界的生命周期

```
Phase 0 结束时: 初始化诚实边界
  来源: Step 0.2质量评级 + Step 0.15素材完整性 + 规则R5的缺口

Phase 1 每个维度结束时: 追加维度级局限
  来源: 每个维度报告的"溯源警告"和"无法溯源的内容"

Phase 2 每个维度结束时: 追加方法级局限
  来源: 每个方法的"失效条件"、"反例"、"可证伪性"、"深度级别"

Phase 3 每个维度结束时: 追加特有局限
  来源: 特有维度的质量评级和溯源覆盖

Phase 4 交叉分析结束时: 追加交叉发现的局限
  来源: 三重交叉发现的矛盾、推导断裂、孤独方法

Phase 5 产物生成时: 诚实边界随产物输出
  诚实边界成为产物的必有 section

动态更新时: 诚实边界随新素材更新
  修复的缺口 → 缩减 / 新发现的缺口 → 扩大
```

## C.3 活诚实边界模板

```markdown
## 诚实边界（活文档） — [人物名]
> 最后更新: YYYY-MM-DD (Phase N)
> 版本: N

### 🔴 严重局限（会显著影响蒸馏质量）
| # | 局限 | 来源 | 影响范围 | 发现Phase |
|---|------|------|---------|---------|
| 1 | [X] | [Phase 0评级/Phase 2反例/...] | [哪些结论不可靠] | Phase N |

### 🟡 中等局限（影响部分维度的可靠性）
...

### 🟢 轻微局限（边缘情况）
...

### 最近更新
| 日期 | Phase | 变更 | 影响 |
|------|-------|------|------|
| YYYY-MM-DD | Phase 3 | 发现XX方法的反例 → 追加中等局限#3 | 该方法的置信度降至"中" |

### 提升路径（按优先级）
| 优先级 | 补什么 | 具体操作 | 预期提升 |
|--------|--------|---------|---------|
| 1 | [X素材] | [怎么获取] | 蒸馏置信度: [X%]→[Y%] |
```

# Part D: 产物质量验证（补缺口#20）

## D.1 v2.0 → v3.0 变化

v2.0的产物质量检查 = 双链终检（只有双链完整度一个维度）。v3.0的产物质量 = 4个维度 × 自动化全量 + 人工抽查。

## D.2 四维产物质量验证

### 维度1: 双链完整度（Step 5.3终检 — 全量自动化）

```
✅ 死链数 = 0
✅ 素材覆盖率 ≥ 70%
✅ 段落级溯源覆盖率: CON≥80%, OPI≥90%, CAS≥80%, SOL≥50%
✅ 孤儿单元 ≤ 5%
✅ 双向性闭合
```

### 维度2: 溯源覆盖率（新增 — 全量自动化）

```
全量扫描产物中所有"此人认为/此人说/此人的【X】是"类型的声明:
- 总声明数: N
- 可追溯到原文(L2+)的声明数: M
- 溯源覆盖率: M/N = X%
- 目标: ≥90%

转述距离分析:
- L0（原文引用）占比: X%
- L1（一次转述）占比: X%
- L2（二次抽象）占比: X%
- L3（综合推断）占比: X% ← 此项必须≤5%（L3过多说明推测太多）
```

### 维度3: 方法论验证度（新增 — 全量统计）

```
统计全部方法论的验证状态:
| 验证项 | 通过数/总数 | 通过率 | 目标 |
|--------|----------|--------|------|
| ≥3验证案例 | N/M | X% | 100% |
| ≥2反例分析 | N/M | X% | ≥80% |
| 推导链合格 | N/M | X% | 100% |
| 有可证伪条件 | N/M | X% | 100% |
| 有深度评级 | N/M | X% | 100% |

总体验证度: 五项通过率的加权平均
- ≥90%: 🟢 高验证度
- 70-90%: 🟡 中验证度
- <70%: 🔴 低验证度 → 产物不能发布
```

### 维度4: 使用有效性（新增 — 人工盲测）

```
盲测协议:
1. 从此人的真实素材中选取N个他被问过的问题（不在训练数据中）
2. 让蒸馏产物回答这些问题
3. 对比蒸馏产物的回答 vs 此人的真实回答
4. 评分标准:
   - 回答框架一致: [X%] — 蒸馏产物是否用了此人会用的思维框架
   - 核心判断一致: [X%] — 蒸馏产物是否得出了此人会得出的结论
   - 语言风格一致: [X%] — 蒸馏产物是否用此人会用的方式表达
   - 总体相似度: 三个维度的加权平均

盲测问题设计:
- 覆盖此人不同的问题类型（如果是诊断型→覆盖13种问题类型各1题）
- 包含"此人的盲区问题"（此人可能回避或不擅长的问题）→ 测试蒸馏产物的诚实边界
- 包含"极端情况"（此人极少遇到的情况）→ 测试蒸馏产物的边界行为

盲测结果:
| 问题ID | 问题类型 | 框架一致 | 判断一致 | 风格一致 | 总体 |
|--------|---------|---------|---------|---------|------|
| T001 | [X] | ✅ | ✅ | ⚠️ | 85% |
| ... | ... | ... | ... | ... | ... |

平均总体相似度: X% → 目标 ≥70%

失败模式分析:
- 在什么类型的问题上蒸馏产物最不准？
- 不准的原因是什么？（素材不足？推导错误？此人本身在此问题上的立场就不一致？）
```

## D.3 产物发布门禁

```
全部门禁通过 → ✅ 可以发布

门禁1: 双链完整度 = 死链0 + 覆盖率≥70% + 段落溯源达标
门禁2: 溯源覆盖率 ≥ 90% + L3占比 ≤ 5%
门禁3: 方法论验证度 ≥ 70% (≥90%为优秀)
门禁4: 盲测总体相似度 ≥ 70%

任一未通过 → ❌ 不能发布 → 返回修复清单
```

## 一句话总结

**v3.0不只是升级标准——是改变蒸馏的工作方式。纠错成为常态（不是意外），交叉分析从平面变立体，诚实边界从一次生成变活文档，产物验证从双链检查变四维评测。框架从"一次性蒸馏工具"变成"持续验证系统"。**
