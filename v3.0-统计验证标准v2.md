# v3.0 统计验证标准 v2

> **目的**：把统计验证从"Phase 2专属"升级为"全Phase覆盖"，新增跨人物验证机制、反例挖掘策略、推导链质量标准、可证伪性检查。
> **覆盖缺口**：#6 Phase 1无统计验证、#7 缺乏跨人物验证、#8 反例系统化收集不足、#9 推导链质量标准缺失、#10 无可证伪性检查

---

## Part 1: Phase 1 统计验证要求（补缺口#6）

### 1.1 核心变更

v2.0中，统计验证（频率≥60%、≥3验证案例、≥2反例）只在Phase 2执行。v3.0将其扩展到 **Phase 1的每个维度**。

### 1.2 逐维度统计要求

| Phase 1 维度 | v2.0要求 | v3.0升级后要求 |
|-------------|---------|--------------|
| **1.1 表达DNA** | 3-5条例句 | **≥10例**；句式分布附百分比（非"为主"）；高频词覆盖全量素材 |
| **1.2 智识谱系** | 显式提及列出 | 每个"明确提及"来源附**引用次数**；推测影响附**独立交叉验证** |
| **1.3 时间线** | 转折点附证据 | 每个转折点标注**前后立场的证据数**；核心信念标注**跨年度重申次数**（≥3次才可称为"核心信念"） |
| **1.4 他者视角** | 合理性评估 | 每条批评标注**来自几个独立来源**（≥2个来源=高可信度）；合理性评估附**评判依据的逻辑** |
| **1.5 内在张力** | 矛盾引用原文 | 每对矛盾标注**双方出现频率对比**（A:17次 vs B:3次 → A是主导）；标注**矛盾的稳定性**（加深中/减弱中/稳定） |
| **1.6 概念体系** | 典型应用≥3个 | 典型应用**≥5个**；CON在素材中出现的**总次数**；首次出现和**末次出现**的SRC-ID（判断是否还在用） |
| **1.7 观点库** | 三级分层 | 每个OPI标注**在不同素材中被重复的次数**；仅出现1次→降权 |
| **1.8 案例库** | 有此人分析 | 每个案例标注此人**重复引用此案例的次数**（≥3次=关键案例）；跨年度引用标注时间跨度 |
| **1.9a 关系网络** | 5种关系类型 | 每条关系标注**强度★★★/★★☆/★☆☆**；关系的**建立依据**（原文/推断） |
| **1.10 身份卡** | 50-100字 | 每句话标注**基于哪个维度的什么证据**（不是自由创作） |
| **1.11 诚实边界** | 5类局限 | 每条局限附**具体数字**（不是"部分不完整"而是"X维度中Y%的结论基于推测"） |

### 1.3 Phase 1 统计验证的最低通过标准

```
每个Phase 1维度完成后，必须通过以下检查才能标记为"完成"：

1. 定量声明有数字支撑（不能只写"经常""多次""大多"）
2. 分类声明有百分比（不能只写"主要是X型"）
3. 频率声明有原始计数（不能只写"高频"）
4. 趋势声明有跨度证据（不能只写"越来越"）
```

## Part 2: 方法论深度评级与跨人物验证（补缺口#7）

### 2.1 四级方法论深度

这是v3.0最重要的新增机制。每个从蒸馏中提取的方法论，必须标注深度级别：

| 深度级别 | 验证要求 | 可以声称 | 示例 |
|---------|---------|---------|------|
| **L0: 行为倾向** | 在1人身上出现≥3次但<60%频率 | "此人的行为倾向" | "鲁大魔有时会打断对方（约40%的情况下）" |
| **L1: 个人规则** | 1人+≥60%频率+≥5案例+推导链+反例分析 | "此人的底层运算规则" | "鲁大魔的'不对称回应律'（85%的情况下不直接回答提问）" |
| **L2: 类型模式** | 同类型≥3人身上都出现（频率各≥60%） | "[诊断型/创作型/...]人物的通用模式" | "诊断型人物普遍表现出'不进入提问者框架'（Jobs/鲁大魔/咨询顾问X都≥70%）" |
| **L3: 跨类型模式** | ≥2种类型≥5人身上都出现 | "人类通用底层思维规则" | "几乎所有高水平思考者都表现出'语言层优先律'（芒格/费曼/Jobs/Naval/Paul Graham都≥65%）" |

### 2.2 深度升级流程

```
L0 → L1: 需要本人物内部验证
  - 增加案例数（从3个到5个）
  - 增加频率统计（从"出现"到"≥60%的相关场景"）
  - 建立推导链（从"描述行为"到"解释为什么"）
  - 完成反例分析

L1 → L2: 需要同类型跨人物验证
  - 在≥3个同类型人物上验证此规则
  - 每个验证人物需要≥3个案例
  - 标注在此类型内的通用度和变异度

L2 → L3: 需要跨类型跨人物验证
  - 在≥2种类型≥5人上验证
  - 排除"仅在此类型/文化/时代成立"的可能性
```

### 2.3 跨人物验证数据库

框架需要维护一个 `跨人物验证矩阵`：

```markdown
| 规则/方法 | 鲁大魔(诊断) | Jobs(产品) | 费曼(学术) | 芒格(哲学) | Naval(投资) | ... |
|----------|-----------|----------|----------|----------|----------|-----|
| 不对称回应律 | ✅ 85% | ✅ 70% | ⚠️ 50% | ✅ 75% | — | ... |
| 最小接触面律 | ✅ 75% | ✅ 80% | ✅ 65% | ✅ 90% | ✅ 70% | ... |
| 语言层优先律 | ✅ 90% | ✅ 85% | ✅ 95% | ✅ 90% | ✅ 80% | ... |
| ... | ... | ... | ... | ... | ... | ... |

图例:
✅ = ≥60% 验证通过（N≥3案例）
⚠️ = 40-60% 不确定
— = 未验证/素材不足
```

### 2.4 诚实边界中的深度标注

每条方法论的产出必须标注当前深度级别和升到下一级需要什么：

```markdown
### 方法1: [名称]
- 深度级别: L1 (个人规则)
- 升到L2需要: 在≥3个诊断型人物上验证（建议: 待补充 [具体人名] 的数据）
- 升到L3需要: 在≥5个跨类型人物上验证（当前已有: Jobs ✅ / 费曼 ⚠️）
```

## Part 3: 反例系统化挖掘策略（补缺口#8）

### 3.1 四条反例挖掘路径

v2.0只要求"找≥2个反例"但没说怎么找。v3.0要求执行以下四条路径：

**路径A: 此人自述失败**
```
搜索关键词: "我错了一次""当时不该用""这不适用于""那次我搞错了"
     "后来发现""回过头看""如果重来一次"
搜索范围: 全部素材
输出: 此人明确承认的方法失效案例
```

**路径B: 逻辑反推**
```
对每个方法，定义其适用条件:
  如果方法A适用于条件X，那么条件非X时方法A应该不适用

逻辑反推:
  1. 定义方法A的适用条件X
  2. 在素材中搜索"非X"的情况
  3. 检查此人在"非X"情况下是否用了方法A → 如果用了→标注为反例
  4. 检查此人在"非X"情况下是否不用方法A → 如果不用且成功了→标注为支持
```

**路径C: 外部批评**
```
从他者视角中提取对此人方法论的批评:
  - 有没有人说过他的某个方法在某种情况下不适用？
  - 有没有人指出过他因为用了某个方法而失败的案例？
  - 有没有学术/行业分析指出类似方法的局限性？
```

**路径D: 竞争假说**
```
对每个方法，提出竞争假说:
  "如果不用方法A，用方法B，是否也能得到同样的结果？"
  "此人用了方法A成功的案例，是否也可以用方法B解释？"
  
  → 如果能用方法B解释→ 标注为"竞争假说"（不一定是反例，但降低了方法A的唯一性）
```

### 3.2 反例质量分级

| 质量 | 定义 | 权重 |
|------|------|------|
| 🟢 直接反例 | 此人明确承认的或可观察到的此方法失效 | 高 — 直接削弱方法的可靠性 |
| 🟡 推理反例 | 逻辑上应该存在但素材中未观察到的反例 | 中 — 提示可能存在盲区 |
| 🟠 竞争假说 | 其他方法也可能解释同样的成功案例 | 低 — 不影响方法本身的有效性 |
| ⚪ 无法确认 | 素材不足以判断是否存在反例 | 诚实边界标注 |

### 3.3 反例完整性声明

每个方法必须附：

```markdown
### 反例完整性
- 直接反例: N个 → [列表+SRC-ID]
- 推理反例: N个 → [列表+逻辑分析]
- 竞争假说: N个 → [列表]
- 反例搜索完成度: [已执行路径]/[4条路径]
- 反例置信度: [高/中/低] — 低=搜索不完整/素材不足
```

## Part 4: 推导链质量标准（补缺口#9）

### 4.1 合格的推导链 = 4个条件

v2.0只要求"有推导链"（如果X→因为Y→所以Z），v3.0新增4个质量条件：

```
条件1: X可直接观察
  X必须是素材中可直接观察到的触发条件，有原文证据（≥L2引用）
  
  不合格: "如果他感到对方不诚实" — 无法直接观察
  合格: "如果对方在描述问题时使用了抽象名词且回避具体数字" — 可直接观察

条件2: Y有独立证据
  Y（此人的核心信念/CON）必须以独立于"此推导链"的方式被验证
  不能因为"X→Z成立"就推导出"Y是原因"→Y需要独立的证据来源

  不合格: Y的证据=同一条推导链
  合格: Y在≥3个不同场景下都有其他证据支撑

条件3: Z可直接观察
  Z必须是素材中可直接观察到的行为结果，有原文证据（≥L2引用）

条件4: 替代解释已排除
  有没有其他解释也能解释"为什么X→Z"？
  如果有→标注替代解释+为什么推导链选Y而不选替代解释
```

### 4.2 推导链质量检查单

```markdown
## 推导链质量检查 — [规则名]

### 推导链: 如果[X] → 因为[Y] → 所以[Z]

| 检查项 | 结果 | 说明 |
|--------|------|------|
| X是否可直接观察？ | ✅/❌ | [证据] |
| Y是否有独立证据？ | ✅/❌ | [≥3个其他场景的证据] |
| Z是否可直接观察？ | ✅/❌ | [证据] |
| 替代解释已排除？ | ✅/❌ | [候选替代解释+排除理由] |
| X→Y的连接是否必然？ | ✅/⚠️ | [为什么是"必然"不是"可能"] |
| Y→Z的连接是否必然？ | ✅/⚠️ | [为什么是"必然"不是"可能"] |

### 推导链质量评级: 🟢 合格 / 🟡 部分合格 / 🔴 不合格
```

### 4.3 不合格推导链的常见模式

| 模式 | 示例 | 为什么不合格 |
|------|------|------------|
| **循环论证** | "他效率高→因为他重视效率→所以他效率高" | Y="重视效率"的唯一证据就是X→Z，没有独立证据 |
| **事后解释** | "他做了A→因为他相信B" | 用结果反推原因，B可能在X发生时不存在 |
| **泛化过度** | "他每次做A→因为人类天生喜欢B" | 从个人行为跳到人类天性，中间缺验证 |
| **缺替代解释** | "他做了A→肯定是出于B" | 没有排除C/D/E也能解释A的可能性 |
| **假必然** | "如果X→必然Y→必然Z" | X→Y和Y→Z都不是必然（可能是概率性的），但用"必然" |

## Part 5: 方法论可证伪性（补缺口#10）

### 5.1 每个方法论必须有可证伪条件

v2.0提取的方法论都是描述性的（"此人在X情况下做Y"），没有定义"什么观察能证明这个方法论是错的"。v3.0要求每个方法论附可证伪条件。

### 5.2 可证伪条件模板

```markdown
### 方法[编号]: [名称]

#### 可证伪性
- **证伪条件**: 如果在≥3次观察到 [触发条件X] 的情况下，此人没有使用 [方法Y]，且无法用 [已标注的失效条件] 解释 → 此方法被证伪
- **证伪观测**: 当前是否已观察到证伪条件？是/否
  - 如果"是"→ 此方法已被证伪，应进入纠错日志
  - 如果"否"→ 标注"当前素材中未观察到证伪条件"
- **证伪难度**: 容易/中等/困难
  - 容易: 只需要观察到X不用Y（触发条件常见）
  - 困难: X极少出现（触发条件罕见）
- **证伪难度对置信度的影响**: 
  - 容易证伪但未被证伪 → 置信度高
  - 困难证伪（X极少出现）→ 置信度中，标注"触发条件罕见，验证不充分"
```

### 5.3 全局可证伪性检查

蒸馏完成后，对全部方法论进行可证伪性审计：

```markdown
## 方法论可证伪性审计

| 方法 | 证伪难度 | 已观测证伪？ | 置信度 |
|------|---------|-----------|--------|
| 方法A | 容易 | 否 | 🟢 高 |
| 方法B | 困难 | 否 | 🟡 中 |
| 方法C | 不可能证伪 | N/A | 🔴 不可证伪→进入诚实边界 |

### 不可证伪的方法（问题）
以下方法无法定义可证伪条件，可能是循环定义或过于模糊:
- [方法X] — 原因: [为什么无法证伪] — 建议: [重新定义/降级/删除]
```

## Part 6: 各Phase统计验证门禁

| 门禁 | 位置 | 条件 | 不通过则 |
|------|------|------|---------|
| Phase 1 统计门禁 | 每个1.x完成后 | 定量有数字、分类有百分比、频率有计数 | 不允许标记为"完成" |
| Phase 2 方法门禁 | 每个2.x完成后 | 频率≥60%+≥3案例+≥2反例+合格推导链+有可证伪条件 | 降级为"行为倾向" |
| Phase 2 规则门禁 | Step 2.2完成后 | ≥5案例+≥2反例+推导链4条件全合格+可证伪 | 降级为"暂定规则" |
| 深度评级门禁 | Phase 4 交叉分析时 | 每个方法标注L0/L1/L2/L3 | 未标注的不进入产物 |
| 可证伪性门禁 | Phase 5 产物生成前 | 全部方法论有可证伪条件 | 不可证伪的进入诚实边界 |
| 反例完整性门禁 | Phase 4 诚实边界汇总时 | 4条反例挖掘路径至少执行2条 | 标注"反例搜索不完整" |

---

## 一句话总结

**v2.0的统计验证是"Phase 2专属"——v3.0的统计验证是"全Phase引擎"。Phase 1也有统计要求，方法论分四级深度（L0-L3），反例有四条挖掘路径，推导链有四个质量条件，每个方法必须有可证伪条件。不再有"不可证伪的真理"。**
