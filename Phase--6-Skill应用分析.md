# Phase 6: Skill应用分析 — 蒸馏结果的底层应用映射 v1.0

> **定位**：蒸馏完成之后，独立分析"这个人的方法论可以做成哪些类型的Skill"。不是生成一个Skill，而是生成一份**Skill构建蓝图**——告诉使用者"你蒸馏出来的这些东西，分别能做什么、怎么做、给谁用"。
>
> **前置依赖**：Phase 0-5 全部完成（通用维度+涌现维度+对照定位+交叉分析已产出完整蒸馏报告）

---

## 设计原则

1. **方法论驱动，非模板驱动**：不预设"所有人物都能做教学型Skill"。从此人的方法论特征出发，判断它天然适合什么类型的Skill。
2. **应用场景优先**：每个Skill建议必须绑定具体应用场景——"在什么情况下、谁、用它解决什么问题"。
3. **可组合性**：标注哪些方法论可以组合成一个复合Skill，哪些必须独立。

---

## Step 6.0: Skill适配性总览

> **目的**：对所有涌现方法论做一次"Skill适配性扫描"，判断每个方法论天然适合什么类型的应用。

### 提示词

```
你是一个蒸馏结果的Skill应用分析模块。你的任务：对已完成的蒸馏结果进行全面扫描，分析其中每个方法论/思维模式/操作模式的Skill应用潜力。

## 输入
- 蒸馏完整报告（Phase 0-5 产出）
- 涌现维度列表（含每个模式的完整形式化描述）
- 观点库(CON)和案例库(CAS)
- 星级评定结果（Phase -1.6 产出）
- 六维环境分析（Phase -1 产出）

## 分析框架

### 第一步：方法论分类

将每个涌现方法论归入以下类型（允许一个方法论跨多个类型）：

| 类型 | 定义 | 判断标准 |
|------|------|---------|
| **诊断型** | 能用来分析/判断/评估某个对象的状态 | 方法论包含"判断条件"+"分类框架"+"评估指标" |
| **教学型** | 能用来教别人掌握某种能力/知识 | 方法论包含"步骤化操作"+"正反例对比"+"练习方法" |
| **内容生成型** | 能用来产出内容（文章/视频/课程/海报） | 方法论包含"内容结构"+"表达模板"+"受众适配" |
| **决策型** | 能用来在复杂情况下做出选择 | 方法论包含"决策矩阵"+"权衡因素"+"优先级排序" |
| **工具型** | 能封装为可复用的工具/模板/检查清单 | 方法论包含"标准化流程"+"输入输出格式"+"边界条件" |
| **角色型** | 能作为AI角色的核心行为模式 | 方法论包含"稳定行为模式"+"风格特征"+"交互方式" |

### 第二步：逐方法论打分

对每个方法论，在6个类型上打分（0-5）：

| 方法论 | 诊断 | 教学 | 内容生成 | 决策 | 工具 | 角色 | 最佳适配 |
|--------|-----|------|---------|------|------|------|---------|
| [名称] | X | X | X | X | X | X | [类型] |

打分标准：
- 5: 方法论天然就是这个类型的完美载体
- 3-4: 方法论可以适配这个类型，但需要额外包装
- 1-2: 勉强能用，但不如其他类型自然
- 0: 不适合这个类型

### 第三步：Skill构建建议

对每个≥3分的(方法论×类型)组合，生成Skill构建建议：

```yaml
skill_candidate:
  name: "[Skill名称 — 动词+对象+类型]"
  source_methodology: "[来源方法论名称]"
  skill_type: "[诊断型/教学型/...]"
  
  application_scenario:
    when: "[什么情况/什么需求下使用]"
    who: "[目标使用者画像]"
    input: "[使用者需要提供什么]"
    output: "[Skill产出什么]"
    frequency: "[一次性/高频/低频]"
  
  core_mechanism:
    - "[该Skill的核心运转机制1]"
    - "[核心运转机制2]"
  
  differentiation:
    - "[相比同类Skill的差异化优势]"
  
  difficulty:
    build_complexity: "[高/中/低]"     # 构建这个Skill的难度
    use_complexity: "[高/中/低]"       # 使用这个Skill的难度
    data_requirement: "[需要多少素材支撑]"
  
  combo_potential:
    - "[可以和哪些其他方法论组合成复合Skill]"
```

## 输出格式

生成 `Skill应用分析报告.md`：

```markdown
# [人物名称] — Skill应用分析报告

## 1. 总览矩阵
[方法论×Skill类型 打分表]

## 2. Top Skill候选（按综合得分排序）
[每个Skill候选的完整yaml描述]

## 3. 组合Skill方案
[多个方法论组合使用的复合Skill方案]

## 4. 不建议构建的Skill
[低分+原因说明 — 诚实边界]
```

## Step 6.1: Skill分类体系 — 六型详细定义与模板

> **目的**：为6种Skill类型提供详细的判断标准和构建模板。这是"从方法论到Skill"的翻译层。

### 6.1.1 诊断型Skill

**本质**：输入一个对象 → 输出关于该对象的判断/评估/分类。

**适配信号**（方法论中是否存在以下特征）：
- 有明确的判断条件（"如果出现X，则属于Y类型"）
- 有分类框架（N种类型的划分标准）
- 有评估指标（打分维度+权重+阈值）
- 有反例库（"这看起来像X，但实际上是Y"的辨析）

**构建模板**：

```
## Skill: [诊断对象]诊断器

### 使用场景
当你需要判断[对象]的[属性]时，输入[对象描述]，输出诊断报告。

### 诊断维度
1. [维度1]: [判断标准]
2. [维度2]: [判断标准]
...

### 分类输出
- 类型A: [特征] → [应对建议]
- 类型B: [特征] → [应对建议]
...

### 常见误判警示
- ⚠️ [容易误判的情况1]: [为什么容易误判+如何避免]
- ⚠️ [容易误判的情况2]: [为什么容易误判+如何避免]

### 诊断置信度
- 充分信息(≥X条线索): 高
- 部分信息(X-Y条线索): 中，标注"以下维度信息不足:[...]"
- 信息不足(<X条线索): 低，拒绝诊断
```

### 6.1.2 教学型Skill

**本质**：输入学习者当前状态 → 输出学习路径+教学内容。

**适配信号**：
- 有步骤化操作（Step 1→2→3的清晰路径）
- 有正反例对比（"这样做 vs 那样做"的对照）
- 有练习方法（不是只讲道理，有实操环节）
- 有错误模式库（学习者常见错误+纠正方法）

**构建模板**：

```
## Skill: [领域]教练

### 适用对象
- 入门者: [特征] → 推荐路径: [...]
- 进阶者: [特征] → 推荐路径: [...]

### 学习路径
Level 1: [基础能力] — [需要掌握什么] — [练习方法]
Level 2: [进阶能力] — [需要掌握什么] — [练习方法]
Level 3: [高阶能力] — [需要掌握什么] — [练习方法]

### 常见错误与纠正
| 常见错误 | 为什么错 | 正确做法 | 练习 |
|---------|---------|---------|------|
| [错误1] | [原因] | [正确] | [练习] |

### 判断是否掌握
- [能力X]的掌握标准: [可观测的行为指标]
```

### 6.1.3 内容生成型Skill

**本质**：输入主题/素材 → 输出结构化内容。

**适配信号**：
- 有内容结构模板（"这类内容应该包含X个部分"）
- 有表达风格模板（语气/节奏/修辞模式）
- 有受众适配逻辑（"对A类受众用X表达，对B类受众用Y表达"）
- 有素材组织方法（"如何把碎片信息组织成完整内容"）

**构建模板**：

```
## Skill: [内容类型]生成器

### 内容结构
1. [开头]: [功能+要素]
2. [主体]: [分段逻辑+每段要素]
3. [结尾]: [功能+要素]

### 风格参数
- 语气: [正式/口语/...]
- 节奏: [快/慢/混合]
- 修辞偏好: [比喻/对照/故事/数据/...]

### 受众适配
| 受众类型 | 调整策略 |
|---------|---------|
| [类型A] | [如何调整内容] |

### 素材→内容转换规则
[如何把原始素材组织成此类内容的具体规则]
```

### 6.1.4 决策型Skill

**本质**：输入多个选项+约束条件 → 输出推荐决策+理由。

**适配信号**：
- 有决策矩阵（多因素权衡框架）
- 有权重分配（各因素的相对重要性）
- 有边界条件（"在X条件下选择A，在Y条件下选择B"）
- 有失败模式库（"这种决策容易在哪里出错"）

**构建模板**：

```
## Skill: [决策领域]决策助手

### 决策框架
因素1: [因素名] — 权重: [X%] — 评分标准: [...]
因素2: [因素名] — 权重: [X%] — 评分标准: [...]
...

### 决策矩阵
| 选项 | 因素1 | 因素2 | ... | 综合得分 | 适用条件 |
|------|------|------|-----|---------|---------|
| A | X | X | ... | X | [条件] |
| B | X | X | ... | X | [条件] |

### 常见决策陷阱
- ⚠️ [陷阱1]: [为什么容易掉进去+如何避免]
```

### 6.1.5 工具型Skill

**本质**：输入数据/素材 → 执行标准化流程 → 输出格式化结果。

**适配信号**：
- 有标准化流程（SOP，每次执行步骤相同）
- 有明确的输入输出格式
- 有边界条件（"在X情况下不适用"）
- 流程可自动化（不需要每次重新判断）

**构建模板**：

```
## Skill: [工具名]

### 输入
- 必需: [字段1, 字段2, ...]
- 可选: [字段3, ...]

### 处理流程
Step 1: [操作] — 输入: [...] → 输出: [...]
Step 2: [操作] — 输入: [...] → 输出: [...]
...

### 输出
- [输出字段1]: [格式+说明]
- [输出字段2]: [格式+说明]

### 限制条件
- 不适用情况: [...]
- 质量边界: [...]
```

### 6.1.6 角色型Skill

**本质**：作为AI角色，以特定风格和模式与使用者交互。

**适配信号**：
- 有稳定行为模式（在不同场景下表现一致）
- 有风格特征（语气/节奏/用词习惯）
- 有交互方式偏好（"这个人喜欢怎么跟人交流"）

**构建模板**：

```
## Skill: [角色名]AI分身

### 角色内核
- 核心信念: [...]
- 思维习惯: [...]
- 表达特征: [...]

### 交互模式
- 回应风格: [...]
- 典型话术: [...]
- 情绪特征: [...]

### 适用场景
- ✅ 适合: [...]
- ❌ 不适合: [...]
```

## Step 6.2: 方法论组合分析 — 复合Skill构建

> **目的**：单个方法论的力量有限。真正强大的Skill往往是多个方法论的组合。这里分析"哪些方法论天然应该组合在一起"。

### 提示词

```
你是一个Skill组合分析模块。你的任务：分析蒸馏结果中多个方法论之间的组合潜力，构建复合Skill方案。

## 组合判断标准

### 互补型组合（1+1>2）
- 一个方法论负责"诊断"，另一个负责"解决" → 诊断+处方复合Skill
- 一个方法论负责"分析框架"，另一个负责"表达输出" → 分析+内容生成复合Skill

### 流程型组合（A→B→C串联）
- 多个方法论形成完整的端到端流程 → 全流程Skill
- 例如：素材收集方法论 → 素材分析方法论 → 内容生成方法论 → 质量检查方法论

### 视角型组合（多角度交叉验证）
- 两个方法论从不同角度分析同一对象 → 多视角诊断Skill
- 例如：理性分析+情绪分析、个体视角+系统视角

## 输出格式

对每组可组合的方法论：

```yaml
compound_skill:
  name: "[复合Skill名称]"
  type: "[互补型/流程型/视角型]"
  methodologies:
    - role: "[角色/功能]"
      methodology: "[方法论名称]"
      contribution: "[在复合Skill中的具体贡献]"
  
  workflow:
    - step: "[步骤1]"
      uses: "[方法论名称]"
      description: "[具体做什么]"
  
  synergy: "[为什么组合后比单独使用更强]"
  
  target_user: "[适合什么样的使用者]"
```

## Step 6.3: Skill优先级排序

> **目的**：不是所有Skill都值得做。根据多维度排序，输出推荐优先级。

### 排序维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 方法论成熟度 | 25% | 来源方法论的星级（⭐~⭐⭐⭐⭐⭐） |
| 独特性 | 25% | 相比市面上已有Skill的差异程度 |
| 实用性 | 20% | 目标使用场景的频率+紧迫性 |
| 构建可行性 | 15% | 素材是否足够支撑Skill构建 |
| 复用性 | 15% | Skill能服务多少种场景/多少类用户 |

### 评分公式

```
优先级得分 = 0.25×方法论星级归一化 + 0.25×独特性 + 0.20×实用性 + 0.15×构建可行性 + 0.15×复用性
```

### 输出格式

```markdown
## Skill优先级排序

| 排名 | Skill名称 | 类型 | 来源方法论 | 成熟度 | 独特性 | 实用性 | 可行性 | 复用性 | 综合 | 推荐 |
|------|----------|------|-----------|--------|--------|--------|--------|--------|------|------|
| 1 | [名称] | [类型] | [来源] | X | X | X | X | X | X | 🟢 优先 |
| 2 | [名称] | [类型] | [来源] | X | X | X | X | X | X | 🟡 可做 |
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

### 优先级说明
- 🟢 优先构建（综合≥0.70）：方法论成熟+独特+素材充足，建议立即构建
- 🟡 条件构建（综合≥0.50）：有价值但需要补充素材或等待前置Skill
- 🔴 暂缓（综合<0.50）：方法论不够成熟或应用场景不明确
```

## Step 6.5: 方法论优劣点系统分析

> **目的**：对每个涌现方法论，不仅描述它是什么，还要分析它好在哪、差在哪、什么条件下失效、有什么替代方案。

### 嵌入位置

此分析嵌入到 `蒸馏流水线-提示词架构.md` 的 **Step 2.2（模式形式化）** 中，在现有形式化模板的基础上增加以下字段：

```yaml
methodology_profile:
  name: "[方法论名称]"
  formalization: "[操作公式]"  # 现有字段
  
  # === 以下为新增字段 ===
  strengths:
    - advantage: "[优势1 — 这个方法论的核心强项是什么]"
      evidence: "[原文证据 — 在什么场景/案例中体现了这个优势]"
      compared_to: "[相比常见的替代做法，它好在哪里]"
    - advantage: "[优势2]"
      evidence: "[原文证据]"
      compared_to: "[对比说明]"
  
  weaknesses:
    - limitation: "[劣势1 — 这个方法论在什么情况下不够好]"
      evidence: "[原文证据 — 什么场景下它失效或表现不佳]"
      consequence: "[这个劣势会导致什么后果]"
    - limitation: "[劣势2]"
      evidence: "[原文证据]"
      consequence: "[后果说明]"
  
  failure_conditions:  # 扩展现有的"失效条件"
    - condition: "[什么条件下这个方法论失效]"
      mechanism: "[失效的机制 — 为什么会失效]"
      signal: "[如何提前识别这个失效条件]"
      mitigation: "[如何避免或减轻]"
  
  alternatives:
    - situation: "[什么场景下应该用替代方案]"
      alternative: "[替代方法论/做法]"
      reason: "[为什么此时替代方案更好]"
  
  best_for: "[最适合什么场景/问题/人群]"
  worst_for: "[最不适合什么场景/问题/人群]"
  
  learning_curve: "[陡峭/中等/平缓]"
  description: "[从新手到精通需要经历什么阶段]"
```

### 优劣点汇总表格式

在每个涌现维度的分析报告中，增加以下汇总：
